# 第三阶段 - 模型服务与部署

> 目标：不依赖 OpenAI，能自己运转一个开源模型服务。
> 核心心智：**把模型服务看作一个"特殊微服务"** —— 高延迟、不稳定、有概率出错。

---

## 1. Java 工程师视角的理解

**把模型服务看作一个"特殊微服务"**：
- 输入：prompt 文本（或 token ids）
- 输出：token 流
- 特殊点：
  - 单次推理耗时秒级
  - 显存占用大
  - 需要 batch 才能提高吞吐

---

## 2. 三大推理框架对比

| 框架 | 出品方 | 强项 | 弱项 | 何时选 |
|------|--------|------|------|--------|
| **vLLM** | UCB | 吞吐量王者（PagedAttention）、生态最广 | 部分小众模型适配慢 | **默认首选** |
| **TGI** | HuggingFace | 与 HF 生态贴合、量化支持好 | 比 vLLM 略慢 | 用 HF 模型时 |
| **TensorRT-LLM** | NVIDIA | 极致延迟、A100/H100 优化最好 | 编译复杂、调试难 | 生产极致优化时 |
| **llama.cpp** | 社区 | CPU/边缘部署、GGUF 量化 | 并发弱 | 没显卡、嵌入式 |
| **Ollama** | 社区 | 易用、Mac 友好 | 生产不推荐 | 本地开发调试 |

---

## 3. PagedAttention 原理（必懂一句话版）

> 类比操作系统的虚拟内存分页。把 KV Cache 切成固定大小的 block，按需分配，消除显存碎片，从而支持超大 batch。

**为什么重要**：这是 vLLM 吞吐量碾压其他框架的根本原因。理解了它就理解了"为什么显存碎片是大敌"。

## 4. 动手路径（按周）

### Week 1：本地体验

```bash
# Mac 用 Ollama 跑通
ollama pull qwen2.5:7b
ollama run qwen2.5:7b
```

用 OpenAI Python SDK 连接 Ollama（默认 `localhost:11434`）。

读 vLLM README，理解启动参数。

### Week 2：云端 GPU 部署

#### 选云平台（按需计费，T4/L4 大概 1~3 元/小时）
- AutoDL（国内最便宜）
- 阿里云
- Lambda
- RunPod

#### 部署 vLLM 服务

```bash
python -m vllm.entrypoints.openai.api_server \
  --model Qwen/Qwen2.5-7B-Instruct \
  --port 8000 \
  --max-model-len 8192
```

用 `curl` 或 Postman 调用 `/v1/chat/completions`：

```bash
curl http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "Qwen/Qwen2.5-7B-Instruct",
    "messages": [{"role": "user", "content": "你好"}],
    "stream": true
  }'
```

### Week 3：Java 客户端集成

- Spring Boot 项目里直接把 `base-url` 指向 vLLM
- **完全复用 OpenAI 客户端代码**
- 体验"模型即微服务"的爽感

## 6. 显存估算公式（必背）

```
显存 ≈ 参数量 × 精度字节数 + KV Cache + 框架开销
```

| 精度 | 每参数字节数 | 7B 模型显存 |
|------|-------------|------------|
| FP16 | 2 | 14 GB |
| INT8 | 1 | 7 GB |
| INT4 | 0.5 | 4 GB |

**额外预留 2-4GB 给 KV Cache 和框架开销**。

## 8. 避坑点

- **显存计算公式**：`参数量 × 精度字节数`。7B FP16 ≈ 14GB，INT8 ≈ 7GB，INT4 ≈ 4GB。还要预留 2-4GB 给 KV Cache。
- **`max-model-len` 调小**：默认 8K，对显存压力大。生产可设 4K，前端做截断。
- **并发不是越高越好**：vLLM 有 `--max-num-seqs`，超了会 OOM。
- **模型下载慢**：用 `HF_ENDPOINT=https://hf-mirror.com` 切国内镜像，或用魔搭社区（ModelScope）。
