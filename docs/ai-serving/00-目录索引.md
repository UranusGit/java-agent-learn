# AI Serving 服务化深耕系列

> 本系列聚焦"**怎么搭平台**"，与 [`../tutorials/spring-ai-2.0/`](../tutorials/spring-ai-2.0/) 的"**怎么用框架**"互补。
>
> 适合：已经能用 Spring AI 写出能跑的 Agent，想往上走做平台 / 中台 / 商业化的人。

---

## 为什么单独成系列

Spring AI 2.0 教程解决"**单机跑通 Agent**"。但企业级 AI 系统还要回答：

- 多家模型（OpenAI / Anthropic / 通义 / 智谱 / DeepSeek）怎么统一管理？
- 万物皆向量，到底选哪个向量库？
- 100 个 Agent、10 个租户，怎么不互相拖死？
- 一天烧 10 万 token，怎么追踪到具体业务？
- 模型一升级效果就崩，怎么回滚？

这些问题不在框架层，在**服务化层**。这就是本系列的定位。

## 学习顺序建议

### 推荐：A → B → C → D → E

按依赖关系：

- A 是基础（没有网关谈不上后续）
- B 服务于 C（Agent 平台一定有 RAG）
- C 是核心产出（综合 A + B）
- D 服务于 C 的商业化（C 上线后才能算账）
- E 是 C 上线后的运维

### 时间紧张：A → C → D → E → B

向量库可以先用 pgvector 顶上，等业务跑起来再回头优化。

### 只学一篇

**Phase A 推理网关**——单独就能产出价值，是所有 AI 中台的入口。

## 阅读约定

- **代码标识**：所有代码块顶部加 `// 本代码仅作学习材料参考`，真正生产代码自己手写。
- **多语言**：以 Java + Spring AI 为主，少量 Python（向量库 SDK / benchmark）。
- **真实数据**：benchmark、成本数据尽量用公开数据；私有数据用脱敏后的占位。
- **跨平台**：默认 macOS / Linux，Windows 路径单独标注。
