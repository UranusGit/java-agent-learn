## 0. 为什么单独搞一个 2.0 目录

Spring AI 2.0 不是 1.0 的小版本升级，而是范式迁移：

| 范式 | 1.0 | 2.0 |
|------|-----|-----|
| Agent Loop | 手写 while + 手动管理 tool calls | `ToolCallingAdvisor` 原生递归迭代 |
| 结构化输出 | `BeanOutputConverter` 转一次 | `StructuredOutputValidationAdvisor` 校验 + 重试 |
| 拦截机制 | `CallAdvisor` / `StreamAdvisor` | 同名但 API 收敛 + 工具链统一 |
| 会话 | `MessageWindowChatMemory` 内存版 | `spring-ai-session` event-sourced 可重放 |
| 工具生态 | `@Tool` 仅本进程 | `@Tool` + `@McpTool` 跨进程跨语言 |
| 可观测 | Micrometer | Micrometer + OTel GenAI Semantic Conventions |

> 旧目录里的"自研 AgentLoop / BeanOutputConverter"在 2.0 都被原生取代。把旧目录当历史，把本目录当未来。

## 2. 三大反复出现的设计原则

整个 2.0 学习会反复强化这三条：

1. **Workflow > Agent**（来自 Anthropic《Building Effective Agents》）—— 能用确定性 DAG 解决的，不要用自主 Agent。10、09 全程体现。
2. **失败要被 LLM 看到**（来自 Claude Code 源码）—— Tool 异常不要抛，要返回 `ToolResult.error(stderr)`，让 LLM 自我修复。04、13 详谈。
3. **三重保护**（maxTurns / 预算 / 死循环检测）—— Agent 必须有终止保护。13 详谈。

## 4. 五大 Workflow 模式学习路径

Anthropic《Building Effective Agents》(2024-12-19) 是企业 Agent 的金科玉律。

```
读 10 理解 5 个模式怎么封装成 Advisor（理论 + API + 综合实战）
   ↓
读 13 知道 5 个 Advisor 怎么防失控（生产化）
   ↓
读 09 知道 5 个 Advisor 撑不住时怎么升级到编排引擎（未来）
```

每个模式都遵循"模式定义 → 适用场景 → 反模式 → Advisor 实现 → 完整 demo → 验收"六段结构。

## 6. 验收总清单

完成全部 30 篇后你应该能：

**入门**：

- [ ] 不查阅资料就能把 1.0 项目升级到 2.0
- [ ] 不用任何 while 循环写出一个 Agent 应用

**工程化核心**：

- [ ] 设计 Advisor 链 order，知道何时用 Base / Call / Stream
- [ ] 用 Reactor Flux 写出健壮的流式 + 工具调用
- [ ] 写出可发布的 MCP Server（含鉴权 + 审计）
- [ ] 处理 PDF/Word/Excel/音频/视频统一抽象

**生产化**：

- [ ] 搭出企业级 RAG（含 Hybrid Search + 评估）
- [ ] 五大 Workflow 模式每个都能在 1 小时内搭出 demo
- [ ] 设计离线 + 在线评估闭环
- [ ] 用 Testcontainers + WireMock 写四层测试
- [ ] 设计 4 层 Prompt Injection 防御 + Agent 防失控三重保护

**架构与反思**：

- [ ] 配出 OTel + Langfuse + Prompt Cache 的生产可观测栈
- [ ] 设计复杂度路由 + 多模型 fallback + 信创合规
- [ ] 设计 Event Sourcing + CQRS 的 AI 原生系统
- [ ] 设计多租户 + 审计 + 配额计费的 Agent 平台
- [ ] 用 ADR 模板做框架选型决策

**深化与综合**：

- [ ] 应用 CoT / ToT / ReAct / Self-Consistency / Reflexion 五大 Prompt 模式
- [ ] 选型 + 微调 embedding 模型（含 LoRA / hard negative）
- [ ] 设计六种记忆（短/长/情节/语义/程序/元）协同架构
- [ ] 跑通 LLM 系统的完整 SRE（SLI/SLO + 漂移检测 + Runbook）
- [ ] 搭建代码/Prompt/模型/数据 四套版本化的 CI/CD
