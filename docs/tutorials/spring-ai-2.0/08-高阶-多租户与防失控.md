> 让 Agent 在多租户环境下安全运行，不爆炸、不越权、可审计。

## 1. 三个核心问题

| 问题 | 后果 | 解决 |
|------|------|------|
| 多租户数据串 | 法律风险 + 用户投诉 | `ToolContext` + 会话隔离 |
| Agent 死循环 / 失控 | 成本爆炸 + 服务不可用 | 三重保护 |
| 没有审计追溯 | 合规失败 + 无法 debug | event-sourced 审计日志 |

## 2. 多租户隔离

### 2.1 三层隔离

| 层 | 隔离手段 | 实现 |
|----|---------|------|
| 会话层 | `conversationId` 区分 | `"tenant-{tenantId}-user-{userId}"` |
| 上下文层 | `ToolContext` 注入 | `Map.of("tenantId", "acme")` |
| 数据层 | Tool 内部按 tenantId 过滤 | `WHERE tenant_id = ?` |

### 2.2 TenantAwareAdvisor

```java
package org.demo02.toolkit.guard;

import org.springframework.ai.chat.client.ChatClientRequest;
import org.springframework.ai.chat.client.ChatClientResponse;
import org.springframework.ai.chat.client.advisor.api.*;
import org.springframework.core.Ordered;
import reactor.core.publisher.Flux;

public class TenantAwareAdvisor implements CallAdvisor, StreamAdvisor {

    @Override
    public String getName() { return "TenantAwareAdvisor"; }

    @Override
    public int getOrder() { return Ordered.HIGHEST_PRECEDENCE + 100; }

    @Override
    public ChatClientResponse adviseCall(ChatClientRequest request, CallAdvisorChain chain) {
        String tenantId = extractTenantId(request);

        ChatClientRequest newReq = request.mutate()
                .context(ctx -> {
                    ctx.put("tenantId", tenantId);
                    ctx.put("conversationId", "tenant-" + tenantId + "-" + extractUserId(request));
                })
                .build();

        return chain.nextCall(newReq);
    }

    @Override
    public Flux<ChatClientResponse> adviseStream(ChatClientRequest request, StreamAdvisorChain chain) {
        String tenantId = extractTenantId(request);
        ChatClientRequest newReq = request.mutate()
                .context(ctx -> ctx.put("tenantId", tenantId))
                .build();
        return chain.nextStream(newReq);
    }

    private String extractTenantId(ChatClientRequest req) {
        // 从请求头 / 用户身份 / API key 拿
        Object tid = req.context().get("tenantId");
        if (tid == null) throw new SecurityException("Missing tenantId");
        return tid.toString();
    }

    private String extractUserId(ChatClientRequest req) {
        return req.context().getOrDefault("userId", "anonymous").toString();
    }
}
```

### 2.3 Tool 内部用 tenantId 过滤

```java
@Tool(description = "查询当前租户的订单")
public List<Order> listOrders(ToolContext context) {
    String tenantId = (String) context.getContext().get("tenantId");
    String userId = (String) context.getContext().get("userId");

    // SQL 必须 WHERE tenant_id
    return orderRepository.findByTenantIdAndUserId(tenantId, userId);
}
```

**关键**：所有数据库查询必须强制带 tenantId，不能让 LLM 自己拼 SQL（避免注入 + 避免跨租户）。

### 2.4 会话隔离

```java
@Bean
ChatMemory chatMemory(SessionRepository sessionRepository) {
    return SessionBackedChatMemory.builder()
            .sessionRepository(sessionRepository)
            .build();
}

@GetMapping("/chat")
public String chat(@RequestParam String q, @RequestHeader("X-Tenant-Id") String tenantId,
                   @RequestHeader("X-User-Id") String userId) {
    String conversationId = "tenant-" + tenantId + "-user-" + userId;
    return chatClient.prompt()
            .user(q)
            .toolContext(Map.of("tenantId", tenantId, "userId", userId))
            .advisors(spec -> spec.param(ChatMemory.CONVERSATION_ID, conversationId))
            .call()
            .content();
}
```

不同租户的 conversationId 永远不同 → 永远不会串数据。

## 3. 三重保护：Agent 不失控

### 3.1 三重保护是什么

| 保护 | 防止 | 默认值 |
|------|------|-------|
| maxTurns | Agent 无限循环 | 10 |
| 预算（token / $） | 单次任务成本爆炸 | 10k token |
| 死循环检测 | 同一 tool call 反复执行 | 3 次相同调用 |

### 3.2 MaxTurnsGuardAdvisor

```java
package org.demo02.toolkit.guard;

import org.springframework.ai.chat.client.ChatClientRequest;
import org.springframework.ai.chat.client.ChatClientResponse;
import org.springframework.ai.chat.client.advisor.api.*;
import org.springframework.core.Ordered;
import reactor.core.publisher.Flux;

public class MaxTurnsGuardAdvisor implements CallAdvisor, StreamAdvisor {

    private final int maxTurns;

    public MaxTurnsGuardAdvisor(int maxTurns) {
        this.maxTurns = maxTurns;
    }

    @Override
    public String getName() { return "MaxTurnsGuardAdvisor"; }

    @Override
    public int getOrder() { return Ordered.HIGHEST_PRECEDENCE + 290; }   // 比 ToolCallingAdvisor 内层

    @Override
    public ChatClientResponse adviseCall(ChatClientRequest request, CallAdvisorChain chain) {
        Integer turn = (Integer) request.context().getOrDefault("currentTurn", 0);
        if (turn >= maxTurns) {
            return ChatClientResponse.builder()
                    .chatResponse(ChatResponse.builder()
                            .withGenerations(List.of(new Generation(
                                    new AssistantMessage("[ABORT] 超过最大轮数 " + maxTurns)
                            )))
                            .build())
                    .build();
        }

        ChatClientRequest newReq = request.mutate()
                .context(ctx -> ctx.put("currentTurn", turn + 1))
                .build();
        return chain.nextCall(newReq);
    }

    @Override
    public Flux<ChatClientResponse> adviseStream(ChatClientRequest request, StreamAdvisorChain chain) {
        ChatClientResponse resp = adviseCall(request, new NoopCallChain());
        return Flux.just(resp);
    }
}
```

### 3.3 TokenBudgetGuardAdvisor

```java
public class TokenBudgetGuardAdvisor implements CallAdvisor {

    private final int maxTokens;
    private final AtomicInteger used = new AtomicInteger(0);

    public TokenBudgetGuardAdvisor(int maxTokens) {
        this.maxTokens = maxTokens;
    }

    @Override
    public String getName() { return "TokenBudgetGuardAdvisor"; }

    @Override
    public int getOrder() { return Ordered.HIGHEST_PRECEDENCE + 280; }

    @Override
    public ChatClientResponse adviseCall(ChatClientRequest request, CallAdvisorChain chain) {
        ChatClientResponse resp = chain.nextCall(request);
        if (resp.chatResponse() != null && resp.chatResponse().getMetadata() != null) {
            Usage usage = resp.chatResponse().getMetadata().getUsage();
            used.addAndGet(usage.getTotalTokens());
            if (used.get() > maxTokens) {
                log.warn("Token 预算超限：{} / {}", used.get(), maxTokens);
            }
        }
        return resp;
    }
}
```

### 3.4 LoopDetectorAdvisor（死循环检测）

```java
public class LoopDetectorAdvisor implements CallAdvisor {

    private static final int MAX_SAME_CALL = 3;

    @Override
    public String getName() { return "LoopDetectorAdvisor"; }

    @Override
    public int getOrder() { return Ordered.HIGHEST_PRECEDENCE + 310; }   // 在 ToolCallingAdvisor 之内

    @Override
    public ChatClientResponse adviseCall(ChatClientRequest request, CallAdvisorChain chain) {
        ChatClientResponse resp = chain.nextCall(request);

        if (resp.chatResponse() != null && resp.chatResponse().hasToolCalls()) {
            String toolCallSignature = resp.chatResponse().getToolCalls().stream()
                    .map(tc -> tc.name() + ":" + tc.arguments())
                    .collect(Collectors.joining("|"));

            @SuppressWarnings("unchecked")
            Map<String, Integer> counter = (Map<String, Integer>) request.context()
                    .computeIfAbsent("toolCallCounter", k -> new HashMap<String, Integer>());

            int count = counter.getOrDefault(toolCallSignature, 0) + 1;
            counter.put(toolCallSignature, count);

            if (count >= MAX_SAME_CALL) {
                log.error("检测到死循环：{} 已调用 {} 次", toolCallSignature, count);
                return abortResponse("检测到重复 tool 调用，已中止");
            }
        }

        return resp;
    }

    private ChatClientResponse abortResponse(String message) {
        return ChatClientResponse.builder()
                .chatResponse(ChatResponse.builder()
                        .withGenerations(List.of(new Generation(new AssistantMessage("[ABORT] " + message))))
                        .build())
                .build();
    }
}
```

## 4. 审计：event-sourced 落库

### 4.1 为什么要审计

- **合规**：金融、医疗、教育等行业要求 AI 决策可追溯
- **debug**：线上出问题能复现
- **改进**：分析 prompt + 结果，持续优化

### 4.2 用 spring-ai-session 自动审计

见 L4 §5。每个 UserMessage / AssistantMessage / ToolCall / ToolResult 都是一个事件，自动落到 DB。

### 4.3 自定义审计 Advisor

```java
public class AuditAdvisor implements CallAdvisor, StreamAdvisor {

    private final AuditEventRepository auditRepo;

    @Override
    public String getName() { return "AuditAdvisor"; }

    @Override
    public int getOrder() { return Ordered.HIGHEST_PRECEDENCE + 90; }   // 比 Tenant 更外层

    @Override
    public ChatClientResponse adviseCall(ChatClientRequest request, CallAdvisorChain chain) {
        String tenantId = (String) request.context().get("tenantId");
        String userId = (String) request.context().get("userId");
        String input = request.prompt().getInstructions().toString();

        long start = System.currentTimeMillis();
        ChatClientResponse response;
        try {
            response = chain.nextCall(request);
        } catch (Exception e) {
            auditRepo.save(AuditEvent.builder()
                    .tenantId(tenantId)
                    .userId(userId)
                    .input(input)
                    .error(e.getMessage())
                    .durationMs(System.currentTimeMillis() - start)
                    .timestamp(Instant.now())
                    .build());
            throw e;
        }

        auditRepo.save(AuditEvent.builder()
                .tenantId(tenantId)
                .userId(userId)
                .input(input)
                .output(response.chatResponse().getResult().getOutput().getText())
                .durationMs(System.currentTimeMillis() - start)
                .timestamp(Instant.now())
                .build());

        return response;
    }

    @Override
    public Flux<ChatClientResponse> adviseStream(ChatClientRequest request, StreamAdvisorChain chain) {
        // 流式版本：聚合后落库
        StringBuilder buffer = new StringBuilder();
        long start = System.currentTimeMillis();
        return chain.nextStream(request)
                .doOnNext(resp -> {
                    String token = resp.chatResponse().getResult().getOutput().getText();
                    if (token != null) buffer.append(token);
                })
                .doOnComplete(() -> auditRepo.save(AuditEvent.builder()
                        .tenantId((String) request.context().get("tenantId"))
                        .userId((String) request.context().get("userId"))
                        .input(request.prompt().getInstructions().toString())
                        .output(buffer.toString())
                        .durationMs(System.currentTimeMillis() - start)
                        .timestamp(Instant.now())
                        .build()));
    }
}
```

### 4.4 AuditEvent 表

```sql
CREATE TABLE audit_event (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    tenant_id VARCHAR(64) NOT NULL,
    user_id VARCHAR(64) NOT NULL,
    input TEXT NOT NULL,
    output TEXT,
    error TEXT,
    duration_ms BIGINT,
    timestamp TIMESTAMP NOT NULL,
    INDEX idx_tenant_timestamp (tenant_id, timestamp),
    INDEX idx_user_timestamp (user_id, timestamp)
);
```

### 4.5 审计查询

```sql
-- 某租户最近 7 天的 AI 调用
SELECT * FROM audit_event
WHERE tenant_id = 'acme'
  AND timestamp > DATE_SUB(NOW(), INTERVAL 7 DAY)
ORDER BY timestamp DESC;

-- 找出失败调用
SELECT * FROM audit_event
WHERE error IS NOT NULL
  AND timestamp > DATE_SUB(NOW(), INTERVAL 1 DAY);

-- 统计平均延迟
SELECT tenant_id, AVG(duration_ms) avg_ms, COUNT(*) cnt
FROM audit_event
WHERE timestamp > DATE_SUB(NOW(), INTERVAL 1 DAY)
GROUP BY tenant_id;
```

## 5. 敏感数据脱敏

### 5.1 在 AuditAdvisor 里脱敏

```java
private String maskSensitive(String text) {
    return text
            .replaceAll("\\b\\d{16,19}\\b", "[CARD]")        // 银行卡号
            .replaceAll("\\b\\d{15}|\\d{18}\\b", "[ID]")      // 身份证
            .replaceAll("\\b1[3-9]\\d{9}\\b", "[PHONE]")      // 手机号
            .replaceAll("\\b[\\w.-]+@[\\w.-]+\\.\\w+\\b", "[EMAIL]");
}
```

### 5.2 Prompt 脱敏（不打到 LLM）

```java
public class PiiMaskingAdvisor implements CallAdvisor {
    @Override
    public ChatClientResponse adviseCall(ChatClientRequest req, CallAdvisorChain chain) {
        String masked = maskSensitive(req.prompt().getInstructions().toString());
        ChatClientRequest newReq = req.mutate()
                .prompt(p -> p.addInstructions(new UserMessage(masked)))
                .build();
        return chain.nextCall(newReq);
    }
}
```

**注意**：脱敏后 LLM 可能无法完成任务，要权衡。

### 6.1 完整执行流程

```
请求进来
  ↓
AuditAdvisor 前置（记录请求）
  ↓
TenantAwareAdvisor 前置（注入 tenantId / userId / conversationId）
  ↓
TokenBudgetGuardAdvisor 前置（检查预算）
  ↓
MaxTurnsGuardAdvisor 前置（检查轮数）
  ↓
MemoryAdvisor 前置（按 conversationId 读历史）
  ↓
LoopDetectorAdvisor 前置
  ↓
ToolCallingAdvisor 前置
  ↓
ChatModel.call()
  ↓
ToolCallingAdvisor 后置（执行工具，可能再调 LLM）
  ↓
LoopDetectorAdvisor 后置（检查重复调用）
  ↓
MemoryAdvisor 后置（写历史）
  ↓
MaxTurnsGuardAdvisor 后置（更新轮数）
  ↓
TokenBudgetGuardAdvisor 后置（更新预算）
  ↓
TenantAwareAdvisor 后置（清理上下文）
  ↓
AuditAdvisor 后置（记录响应）
  ↓
返回给用户
```

## 7. 安全检查清单

### 7.1 上线前必查

- [ ] 所有数据库查询都带 `tenant_id` 过滤
- [ ] Tool 方法签名不允许 LLM 自己拼 SQL / 命令
- [ ] `ToolContext` 强制注入 tenantId，缺失时直接抛错
- [ ] `conversationId` 包含 tenantId 前缀
- [ ] 三重保护（maxTurns / 预算 / 死循环）全部到位
- [ ] 审计日志覆盖所有调用（含失败）
- [ ] 敏感字段（PII）在审计前脱敏
- [ ] OTel trace 也包含 tenantId 标签
- [ ] API 调用走 HTTPS + API key 鉴权

### 7.2 Tool 黑名单

以下工具**绝不**让 LLM 自由调用：
- 直接执行 SQL（用预定义的查询方法代替）
- 执行 shell 命令
- 文件系统任意读写
- 网络请求（白名单域名）
- 邮件 / 短信发送（必须有审批流）
