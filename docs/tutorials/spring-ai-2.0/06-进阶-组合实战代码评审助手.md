## 1. 项目目标

### 1.1 一句话

输入一个 Java 文件的代码 → 输出专业评审报告（bug + 风格 + 安全 + 改进建议）。

### 1.2 为什么用五大模式组合

| 模式 | 怎么用 |
|------|-------|
| **Routing** | 按代码类型路由（Controller / Service / Repository） |
| **Parallelization** | 并行评审 bug / 风格 / 安全 三视角 |
| **Orchestrator-Workers** | 文件超过 N 行时按方法拆分评审 |
| **Evaluator-Optimizer** | 评审报告质量不达标则改进 |
| **Prompt Chaining** | 总流程：分类 → 评审 → 聚合 → 优化 |

### 1.3 为什么不用单一 Agent

| 维度 | 自主 Agent | 五大模式组合 |
|------|-----------|------------|
| 稳定性 | LLM 可能跑偏 | 每步确定 |
| 成本 | Token 不可控 | 每步有预算 |
| 可观测 | 黑盒 | 每步独立 trace |
| 准确率 | 看运气 | Voting + 评估提升 |

### 3.2 Routing Advisor：代码类型路由

```java
package org.demo02.project.review.advisor;

import org.demo02.toolkit.workflow.RoutingAdvisor;
import org.springframework.ai.chat.client.ChatClient;
import org.springframework.beans.factory.annotation.Qualifier;
import org.springframework.context.annotation.*;

@Configuration
public class CodeTypeRouterConfig {

    @Bean("controllerReviewer") ChatClient controllerReviewer(ChatClient.Builder b) {
        return b.defaultSystem("专门评审 Controller，关注 REST 规范、参数校验、HTTP 状态码").build();
    }

    @Bean("serviceReviewer") ChatClient serviceReviewer(ChatClient.Builder b) {
        return b.defaultSystem("专门评审 Service，关注事务边界、业务逻辑、依赖注入").build();
    }

    @Bean("repositoryReviewer") ChatClient repositoryReviewer(ChatClient.Builder b) {
        return b.defaultSystem("专门评审 Repository，关注 SQL 性能、索引、N+1 问题").build();
    }

    @Bean
    RoutingAdvisor codeTypeRouter(
            @Qualifier("classifier") ChatClient classifier,
            @Qualifier("controllerReviewer") ChatClient controller,
            @Qualifier("serviceReviewer") ChatClient service,
            @Qualifier("repositoryReviewer") ChatClient repo,
            @Qualifier("bugReviewer") ChatClient fallback
    ) {
        return RoutingAdvisor.builder()
                .classifier(classifier)
                .routes(java.util.Map.of(
                        "CONTROLLER", controller,
                        "SERVICE", service,
                        "REPOSITORY", repo
                ))
                .fallback(fallback)
                .build();
    }
}
```

### 3.6 CodeReviewService：编排

```java
package org.demo02.project.review;

import org.springframework.ai.chat.client.ChatClient;
import org.springframework.stereotype.Service;

@Service
public class CodeReviewService {

    private final ChatClient chatClient;
    private final RoutingAdvisor router;
    private final ParallelizationAdvisor parallel;
    private final OrchestratorWorkersAdvisor chunking;
    private final EvaluatorOptimizerAdvisor quality;

    public CodeReviewService(
            ChatClient chatClient,
            RoutingAdvisor router,
            ParallelizationAdvisor parallel,
            OrchestratorWorkersAdvisor chunking,
            EvaluatorOptimizerAdvisor quality
    ) {
        this.chatClient = chatClient;
        this.router = router;
        this.parallel = parallel;
        this.chunking = chunking;
        this.quality = quality;
    }

    public String review(String code) {
        // 流程：路由 → 并行评审 → 质量优化
        return chatClient.prompt()
                .user(code)
                .advisors(
                        router.withOrder(Ordered.HIGHEST_PRECEDENCE + 500),
                        parallel.withOrder(Ordered.HIGHEST_PRECEDENCE + 550),
                        quality.withOrder(Ordered.HIGHEST_PRECEDENCE + 580)
                )
                .call()
                .content();
    }

    public String reviewLargeFile(String code) {
        // 大文件：先按方法拆分（Orchestrator-Workers），再走标准流程
        return chatClient.prompt()
                .user(code)
                .advisors(
                        chunking.withOrder(Ordered.HIGHEST_PRECEDENCE + 480),
                        router.withOrder(Ordered.HIGHEST_PRECEDENCE + 500),
                        parallel.withOrder(Ordered.HIGHEST_PRECEDENCE + 550),
                        quality.withOrder(Ordered.HIGHEST_PRECEDENCE + 580)
                )
                .call()
                .content();
    }
}
```

### 3.7 Controller

```java
package org.demo02.project.review;

import org.springframework.web.bind.annotation.*;

@RestController
@RequestMapping("/review")
public class CodeReviewController {

    private final CodeReviewService service;

    public CodeReviewController(CodeReviewService service) {
        this.service = service;
    }

    @PostMapping
    public String review(@RequestBody String code) {
        int lines = code.split("\n").length;
        return lines > 200 ? service.reviewLargeFile(code) : service.review(code);
    }
}
```

## 4. 执行流程示例

输入一段 50 行的 Controller 代码：

```
Step 1: RoutingAdvisor
  classifier LLM 分析 → "CONTROLLER"
  路由到 controllerReviewer

Step 2: ParallelizationAdvisor（在 controllerReviewer 后）
  bug / style / security 三个 LLM 并行跑
  各自输出独立评审结果

Step 3: EvaluatorOptimizerAdvisor
  aggregator LLM 整合三份报告
  reportEvaluator LLM 评估
  不达标则反馈 → aggregator 重写
  最多 2 次循环

Step 4: 输出最终 markdown 报告
```

## 6. 加上三重保护（防失控）

参考 L8 的三重保护，给评审服务加上限：

```java
@Service
public class CodeReviewService {

    private static final int MAX_TOKENS_PER_REVIEW = 50_000;
    private static final int MAX_DURATION_MS = 60_000;

    public String review(String code) {
        long start = System.currentTimeMillis();

        String result = chatClient.prompt()
                .user(code)
                .advisors(
                        router,
                        parallel,
                        quality,
                        // 新增：预算保护
                        new TokenBudgetGuardAdvisor(MAX_TOKENS_PER_REVIEW),
                        // 新增：超时保护
                        new TimeoutGuardAdvisor(MAX_DURATION_MS, start)
                )
                .call()
                .content();

        return result;
    }
}
```

## 7. 加上工具（增强评审）

### 7.1 静态分析工具

```java
@Component
public class CodeAnalysisTools {

    @Tool(description = "用 PMD 检查 Java 代码风格违规")
    public List<Violation> runPmd(@ToolParam("源代码") String code) {
        // 调用 PMD 库
        return pmdRunner.run(code);
    }

    @Tool(description = "解析 AST，提取所有方法签名")
    public List<MethodSignature> extractMethods(@ToolParam("源代码") String code) {
        return JavaParser.parse(code).findAll(MethodDeclaration.class)
                .stream()
                .map(this::toSignature)
                .toList();
    }

    @Tool(description = "查询项目编码规范知识库")
    public String queryStandard(@ToolParam("查询关键词") String keyword) {
        return vectorStore.similaritySearch(keyword).stream()
                .map(Document::getText)
                .collect(Collectors.joining("\n"));
    }
}
```

LLM 可以自主调用这些工具增强评审深度。

## 8. 性能与成本

### 8.1 一次评审的成本（粗算）

| 步骤 | 调用次数 | 单次 token | 总 token |
|------|---------|-----------|---------|
| Classifier | 1 | ~500 | 500 |
| ParallelReview | 3 | ~2000 | 6000 |
| Aggregator | 1 | ~3000 | 3000 |
| ReportEvaluator | 1 | ~1500 | 1500 |
| Aggregator 重写（最坏） | 1 | ~3000 | 3000 |
| **合计（平均）** | | | ~11000 |
| **合计（最坏）** | | | ~14000 |

DeepSeek-Chat 价格 $0.14/M input → 一次评审约 $0.002。可接受。

### 8.2 加速技巧

- 用 `MessageWindowChatMemory` 不要（评审是一次性的）
- 开 Prompt Cache（重复 system prompt 能省 30%+）
- bug / style / security 并行已经做到了，不需要更多优化

## 9. 常见错误

### 9.1 评审报告内容为空

**原因**：某个 Advisor 抛异常被吞了。

**排查**：开 DEBUG 日志看每步输入输出。

### 9.2 大文件拆分错乱

**原因**：Orchestrator-Workers 的 LLM 拆分不合理（拆出空任务）。

**解决**：在 prompt 里强约束"至少拆 2 个子任务，每个子任务必须有 description"。

### 9.3 评审一直循环不结束

**原因**：Evaluator 永远不通过。

**解决**：`maxIterations` 必须 < 3。
