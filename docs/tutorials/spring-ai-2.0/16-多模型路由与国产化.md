## 1. 为什么要做多模型路由

### 1.1 成本差异数量级

| 模型 | 输入 ($/1M token) | 输出 ($/1M token) | 备注 |
|------|-------------------|-------------------|------|
| Claude Opus 4 | 15 | 75 | 最强，最贵 |
| GPT-4 Turbo | 10 | 30 | 综合强 |
| Claude Sonnet | 3 | 15 | 性价比 |
| DeepSeek-V3 | 0.27 | 1.1 | 国产之光，便宜 |
| Qwen2.5-Max | 1.4 | 5.6 | 国产 |
| GPT-4o mini | 0.15 | 0.6 | 便宜 |

**结论**：同一问题，不同模型成本可以差 100 倍。

### 1.2 业务请求的复杂度分布

实测某客服系统 100 万条 query：

| 复杂度 | 占比 | 例子 |
|--------|------|------|
| 简单（FAQ） | 65% | "营业时间"、"退货政策" |
| 中等（多轮） | 25% | "我的订单到哪了" |
| 复杂（推理） | 8% | "对比 A 和 B 产品的差异" |
| 极复杂（多步推理） | 2% | "根据使用历史推荐定制方案" |

65% 用便宜模型 → 成本下降约 60%，质量几乎不变。

## 3. 复杂度路由的实现

### 3.1 复杂度分类器

```java
// org.demo02.routing.ComplexityClassifier
// 本代码仅作学习材料参考

@Component
public class ComplexityClassifier {

    private final ChatClient classifierClient;   // 用最便宜的模型

    public enum Complexity { SIMPLE, MEDIUM, COMPLEX }

    public Complexity classify(String query) {
        String prompt = """
            判断以下用户问题的复杂度，只输出一个词：simple / medium / complex
            
            判断标准：
            - simple：单步、明确、能用一句话回答（如"营业时间"）
            - medium：需要解释、对比、推理 1-2 步
            - complex：多步推理、需要规划、代码生成、深度分析
            
            用户问题：%s
            """.formatted(query);

        String result = classifierClient.prompt()
                .user(prompt).call().content().trim().toLowerCase();

        return switch (result) {
            case "simple" -> Complexity.SIMPLE;
            case "medium" -> Complexity.MEDIUM;
            default -> Complexity.COMPLEX;
        };
    }
}
```

### 3.2 路由表

```java
@Component
public class ModelRouter {

    private final ComplexityClassifier classifier;
    private final Map<Complexity, ChatModel> models;

    public ChatModel route(String query) {
        Complexity c = classifier.classify(query);
        ChatModel model = models.getOrDefault(c, models.get(Complexity.COMPLEX));
        log.info("Query routed: {} -> {} -> {}",
                truncate(query, 50), c, getModelName(model));
        return model;
    }
}
```

## 4. LLM Gateway

### 4.1 自建最小 Gateway

```java
// org.demo02.gateway.LlmGateway
// 本代码仅作学习材料参考

@RestController
@RequestMapping("/demo02/gateway")
public class LlmGateway {

    private final ModelRouter router;
    private final MeterRegistry meters;
    private final QuotaService quota;

    @PostMapping("/chat")
    public ChatResponse chat(@RequestBody ChatRequest req,
                              @RequestHeader("X-Tenant") String tenant,
                              @RequestHeader("X-User") String user) {
        // 1. 配额检查
        if (!quota.tryConsume(tenant, user, 1000)) {
            throw new ResponseStatusException(TOO_MANY_REQUESTS, "Quota exceeded");
        }

        // 2. 路由
        ChatModel model = router.route(req.query());

        // 3. 调用（带 fallback）
        try {
            ChatResponse resp = callWithFallback(model, req);
            
            // 4. 计费
            int tokens = resp.getMetadata().getUsage().getTotalTokens();
            BigDecimal cost = pricingCalculator.compute(model, tokens);
            billingService.charge(tenant, user, cost);
            
            // 5. 指标
            meters.timer("ai.gateway.latency", "model", modelName(model))
                  .record(Duration.ofMillis(req.elapsed()));
            meters.counter("ai.gateway.tokens", "model", modelName(model))
                  .increment(tokens);
            
            return resp;
        } catch (Exception e) {
            meters.counter("ai.gateway.errors", "model", modelName(model)).increment();
            throw e;
        }
    }

    private ChatResponse callWithFallback(ChatModel primary, ChatRequest req) {
        try {
            return callOnce(primary, req);
        } catch (Exception e) {
            log.warn("Primary model failed, falling back: {}", e.getMessage());
            ChatModel fallback = router.fallbackFor(primary);
            return callOnce(fallback, req);
        }
    }
}
```

### 5.2 实时熔断

```java
@Component
public class QuotaGuard {

    public void check(String tenantId, int estimatedTokens) {
        if (!quotaService.tryConsume(tenantId, "llm_tokens", estimatedTokens)) {
            throw new QuotaExceededException("Daily token quota exceeded");
        }
        if (!quotaService.tryConsume(tenantId, "cost_usd",
                pricingCalculator.estimate(estimatedTokens))) {
            throw new QuotaExceededException("Daily cost quota exceeded");
        }
    }
}
```

### 7.2 处理 R1 的 reasoning_content

R1 返回两部分：`reasoning_content`（推理过程）+ `content`（最终答案）：

```java
public R1Result callWithReasoning(String prompt) {
    ChatResponse resp = chatClient.prompt()
            .options(OpenAiChatOptions.builder().model("deepseek-reasoner").build())
            .user(prompt).call().chatResponse();

    String content = resp.getResult().getOutput().getText();
    String reasoning = resp.getResult().getOutput().getMetadata()
            .get("reasoning_content").toString();

    return new R1Result(reasoning, content);
}
```

**前端显示**：UI 折叠显示 reasoning（"思考过程"），让用户看到模型怎么推理。

## 12. 国产化合规

### 12.1 数据出境

《个人信息保护法》《数据安全法》：

- 个人信息不得随意出境。
- 重要数据出境要安全评估。
- 关键行业（金融、医疗、政务）严格限制。

**实际操作**：用国内模型 / 自托管 / Azure OpenAI（中国由世纪互联运营）。

### 12.2 信创目录

政府采购、国企信息化项目要求：

- 硬件：国产 CPU（鲲鹏/海光/飞腾）/ GPU（昇腾）
- OS：麒麟 / 统信
- DB：达梦 / 人大金仓 / GaussDB
- LLM：自托管 Qwen / DeepSeek / ChatGLM

### 12.3 算法备案

国内对外提供 AI 服务需要：

- **算法备案**（网信办）
- **大模型备案**
- **生成内容合规**（内容审核）

### 12.4 内容安全

```java
@Component
public class ContentSafetyAdvisor implements BaseAdvisor {

    private final ContentSafetyClient safety;

    @Override
    public ChatClientResponse after(ChatClientResponse resp, AdvisorChain chain) {
        String text = resp.chatResponse().getText();
        ContentSafetyResult result = safety.check(text);
        
        if (!result.pass()) {
            metrics.counter("ai.content.blocked").increment();
            return overrideResponse(resp, "抱歉，该内容无法显示");
        }
        return resp;
    }
}
```

接入阿里绿网、网易易盾、火山引擎等内容安全 API。

## 13. 实战避坑

### 13.1 "DeepSeek 调用慢"

高峰期紧张。加 fallback（通义 / 自托管）+ 重试退避 + 企业版 API。

### 13.2 "通义协议和 OpenAI 不完全一致"

用 `spring-ai-alibaba-starter`（专门适配）；或用通义的 OpenAI 兼容模式。

### 13.3 "自托管模型 function calling 不支持"

用支持 function calling 的版本（如 Qwen2.5-Instruct）；vLLM 加 `--enable-auto-tool-choice --tool-call-parser hermes`。

### 13.4 "国产模型中文 token 计数不准"

用模型官方 tokenizer（如 `tokenizers` for Qwen）；不要信"1 token = 1.5 字符"的估算。

### 13.5 "昇腾适配版本滞后"

等适配；或用 NVIDIA + 私有化；或用上一代模型。

### 13.6 "级联路由延迟翻倍"

设短超时：便宜模型 2s 不返回就升级；用流式边输出边判断。
