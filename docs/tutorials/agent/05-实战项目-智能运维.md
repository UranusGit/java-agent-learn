### 4.2 K8sClient 封装

```java
@Component
public class K8sClient {

    private final KubernetesClient client;

    public K8sClient() {
        this.client = new KubernetesClient();  // 自动读 ~/.kube/config
    }

    public DeploymentStatus getDeployment(String ns, String name) {
        var dep = client.apps().deployments()
                .inNamespace(ns)
                .withName(name)
                .get();
        if (dep == null) return null;
        return new DeploymentStatus(
            name, ns,
            dep.getStatus().getReplicas(),
            dep.getStatus().getReadyReplicas(),
            dep.getStatus().getAvailableReplicas()
        );
    }

    public List<PodInfo> getPods(String ns, String labelSelector) {
        return client.pods()
                .inNamespace(ns)
                .withLabel(labelSelector.split("=")[0], labelSelector.split("=")[1])
                .list()
                .getItems()
                .stream()
                .map(pod -> new PodInfo(
                    pod.getMetadata().getName(),
                    pod.getStatus().getPhase(),
                    pod.getStatus().getPodIP()
                ))
                .toList();
    }

    public void restartDeployment(String ns, String name) {
        client.apps().deployments()
                .inNamespace(ns)
                .withName(name)
                .rolling().restart();
    }
}
```

### 4.3 Tool 包装

```java
@Component
@RequiredArgsConstructor
public class K8sTools {

    private final K8sClient k8s;

    @Tool("""
        查询指定命名空间下 Deployment 的副本状态。
        使用场景：用户问"几个副本"、"实例数"、"Pod 状态"。
        返回：DeploymentStatus（desired, ready, available）
        """)
    public DeploymentStatus getDeploymentStatus(
            @ToolParam("命名空间，默认 default", example = "default") String namespace,
            @ToolParam("Deployment 名称") String name
    ) {
        DeploymentStatus s = k8s.getDeployment(namespace, name);
        if (s == null) {
            throw new RuntimeException(
                "Deployment " + namespace + "/" + name + " 不存在。" +
                "请确认名称是否正确。"
            );
        }
        return s;
    }

    @Tool("查询指定标签的 Pod 列表")
    public List<PodInfo> getPods(
            @ToolParam("命名空间") String namespace,
            @ToolParam("标签选择器，格式 app=name") String labelSelector
    ) {
        return k8s.getPods(namespace, labelSelector);
    }

    @Tool("""
        重启指定 Deployment（滚动重启）。
        危险操作！调用前必须向用户确认。
        """)
    public String restartDeployment(
            @ToolParam("命名空间") String namespace,
            @ToolParam("Deployment 名称") String name,
            @ToolParam("用户确认标识，必须是 'CONFIRM'") String confirmation
    ) {
        if (!"CONFIRM".equals(confirmation)) {
            throw new RuntimeException("危险操作未确认，请向用户询问是否确认");
        }
        k8s.restartDeployment(namespace, name);
        return "已触发 " + namespace + "/" + name + " 的滚动重启";
    }
}
```

### 4.4 关键设计：危险操作的二次确认

`restartDeployment` 强制要求 `confirmation` 参数为 `CONFIRM`。
LLM 无法自己编出这个值，必须从用户那里得到明确确认。

```
用户："重启 user-service"
LLM：我看到这个操作需要确认。你确定要重启吗？请回复"确认"
用户："确认"
LLM：调用 restartDeployment(..., confirmation="CONFIRM")
```

## 5. PromTools

### 5.1 PromClient

```java
@Component
public class PromClient {

    private final WebClient client;

    public PromClient(@Value("${prometheus.url}") String url) {
        this.client = WebClient.builder().baseUrl(url).build();
    }

    public double query(String promql) {
        return client.get()
                .uri(uri -> uri.path("/api/v1/query")
                        .queryParam("query", promql).build())
                .retrieve()
                .bodyToMono(String.class)
                .timeout(Duration.ofSeconds(5))
                .map(this::extractValue)
                .block();
    }

    private double extractValue(String json) {
        // 简化：解析 PromQL 响应
        // 真实实现用 Jackson
        return 0.0;
    }
}
```

### 5.2 Tool 包装

```java
@Component
@RequiredArgsConstructor
public class PromTools {

    private final PromClient prom;

    @Tool("""
        查询 Prometheus 指标。
        使用场景：用户问 CPU、内存、QPS、错误率等。
        参数必须是合法 PromQL 表达式。
        常用示例：
        - CPU 使用率：rate(container_cpu_usage_seconds_total{pod=~"xxx.*"}[5m])
        - 内存：container_memory_usage_bytes{pod=~"xxx.*"}
        - QPS：rate(http_requests_total[5m])
        """)
    public double queryMetric(
            @ToolParam("PromQL 查询表达式") String promql
    ) {
        try {
            return prom.query(promql);
        } catch (Exception e) {
            throw new RuntimeException(
                "PromQL 执行失败：" + e.getMessage() + "。" +
                "请检查查询表达式是否正确。"
            );
        }
    }
}
```

### 5.3 让 LLM 写 PromQL 的难点

LLM 不一定能写对复杂的 PromQL。解决方法：
1. Tool 描述里给示例（如上）
2. 加一个 `getMetricsHelp` Tool 列出所有可用指标

```java
@Tool("列出常用的 Prometheus 指标名和含义")
public String listAvailableMetrics() {
    return """
        - container_cpu_usage_seconds_total: CPU 使用
        - container_memory_usage_bytes: 内存使用
        - http_requests_total: HTTP 请求总数
        - http_request_duration_seconds: HTTP 耗时
        """;
}
```

## 7. 系统提示词

`prompts/ops-system.st`：
```
你是公司内部的智能运维助手，负责 K8s 集群、Prometheus 指标、应用日志的查询和管理。

【能力】
- 查询 Deployment / Pod 状态
- 查询 Prometheus 指标
- 查询应用日志
- 执行重启等运维操作（需用户确认）

【原则】
1. 查询类操作直接做
2. 危险操作（重启、删除）必须先向用户确认
3. 找不到资源时如实告知，不要编造
4. 多步操作时清晰说明每一步
5. 给出指标时附上简短解读（"CPU 65% 偏高"）

【示例】
用户：用户服务 CPU 高不高？
  → 调 queryMetric('rate(container_cpu_usage_seconds_total{pod=~"user-service.*"}[5m])')
  → 根据返回值判断高低
  → 回答："CPU 使用率 78%，偏高，建议排查"
```

## 10. 关键工程问题

### 10.1 鉴权

> Spring AI 1.0.0 API（`adviseCall` / `ChatClientRequest` / `req.context()` / `chain.nextCall`）。

```java
public class AuthAdvisor implements CallAdvisor {
    @Override
    public String getName() { return "AuthAdvisor"; }

    @Override
    public int getOrder() { return -200; }

    @Override
    public ChatClientResponse adviseCall(ChatClientRequest req, CallAdvisorChain chain) {
        String role = (String) req.context().get("role");
        if (!"ops".equals(role)) {
            throw new RuntimeException("权限不足：需要 ops 角色");
        }
        return chain.nextCall(req);
    }
}
```

### 10.2 审计

```java
public class AuditAdvisor implements CallAdvisor {
    private final AuditRepo repo;

    @Override
    public String getName() { return "AuditAdvisor"; }

    @Override
    public int getOrder() { return 100; }  // 后处理位置

    @Override
    public ChatClientResponse adviseCall(ChatClientRequest req, CallAdvisorChain chain) {
        ChatClientResponse resp = chain.nextCall(req);
        repo.save(new AuditLog(
            (String) req.context().get("user"),
            req.prompt().getUserMessage().getText(),
            LocalDateTime.now()
        ));
        return resp;
    }
}

## 12. 常见问题

### 12.1 LLM 写不对 PromQL

**解决**：
- Tool 描述里给示例
- 加 `listAvailableMetrics` Tool
- 对常见指标提供模板

### 12.2 K8s API 慢

**解决**：
- 加缓存（如 Deployment 状态缓存 30 秒）
- 用 `withTimeout(Duration.ofSeconds(5))`

### 12.3 鉴权复杂

**解决**：
- 用 Spring Security 拿当前用户
- Tool 内部用 `ToolContext` 拿用户信息

### 12.4 LLM 乱重启

**解决**：
- `restartDeployment` 必须 confirmation="CONFIRM"
- 重启 Tool 单独鉴权（admin）
- 加审计日志
