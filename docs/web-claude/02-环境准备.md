### 1.3 WebSocket 配置

新建 `src/main/java/org/demo02/webclaude/ws/WebSocketConfig.java`：

```java
// 本代码仅作学习材料参考
package org.demo02.webclaude.ws;

import org.springframework.context.annotation.Configuration;
import org.springframework.web.socket.config.annotation.EnableWebSocket;
import org.springframework.web.socket.config.annotation.WebSocketConfigurer;
import org.springframework.web.socket.config.annotation.WebSocketHandlerRegistry;

@Configuration
@EnableWebSocket
public class WebSocketConfig implements WebSocketConfigurer {

    private final SessionWebSocketHandler handler;

    public WebSocketConfig(SessionWebSocketHandler handler) {
        this.handler = handler;
    }

    @Override
    public void registerWebSocketHandlers(WebSocketHandlerRegistry registry) {
        registry.addHandler(handler, "/ws/session")
                .setAllowedOrigins("*");
    }
}
```

### 1.4 最简 WS Handler（ping/pong）

新建 `src/main/java/org/demo02/webclaude/ws/SessionWebSocketHandler.java`：

```java
// 本代码仅作学习材料参考
package org.demo02.webclaude.ws;

import org.springframework.stereotype.Component;
import org.springframework.web.socket.CloseStatus;
import org.springframework.web.socket.TextMessage;
import org.springframework.web.socket.WebSocketSession;
import org.springframework.web.socket.handler.TextWebSocketHandler;

import java.util.Map;
import java.util.concurrent.ConcurrentHashMap;

@Component
public class SessionWebSocketHandler extends TextWebSocketHandler {

    private final Map<String, WebSocketSession> sessions = new ConcurrentHashMap<>();

    @Override
    public void afterConnectionEstablished(WebSocketSession session) {
        sessions.put(session.getId(), session);
        System.out.println("[WS] connected: " + session.getId());
    }

    @Override
    protected void handleTextMessage(WebSocketSession session, TextMessage message) {
        String payload = message.getPayload();
        if ("ping".equals(payload)) {
            session.sendMessage(new TextMessage("pong"));
        }
    }

    @Override
    public void afterConnectionClosed(WebSocketSession session, CloseStatus status) {
        sessions.remove(session.getId());
        System.out.println("[WS] closed: " + session.getId());
    }
}
```

### 2.3 WebSocket 客户端

`src/api/ws.ts`：

```typescript
// 本代码仅作学习材料参考
export type WSMessage =
  | { type: 'ping' }
  | { type: 'pong' }
  | { type: 'state'; session_id: string; transition: string; message: any; seq: number }
  | { type: 'error'; reason: string };

export class SessionWS {
  private ws: WebSocket | null = null;
  private lastSeq = 0;
  private listeners: ((msg: WSMessage) => void)[] = [];

  constructor(private url: string) {}

  connect(onOpen?: () => void) {
    this.ws = new WebSocket(this.url);
    this.ws.onopen = () => onOpen?.();
    this.ws.onmessage = (e) => {
      const msg: WSMessage = JSON.parse(e.data);
      this.listeners.forEach((l) => l(msg));
    };
    this.ws.onclose = () => this.reconnect();
  }

  send(msg: WSMessage) {
    this.ws?.send(JSON.stringify(msg));
  }

  onMessage(l: (msg: WSMessage) => void) {
    this.listeners.push(l);
  }

  reconnect() {
    setTimeout(() => this.connect(), 1000);
  }
}
```

### 2.4 主 App

`src/App.tsx`：

```tsx
// 本代码仅作学习材料参考
import { useEffect, useState } from 'react';
import { SessionWS } from './api/ws';

export default function App() {
  const [status, setStatus] = useState<'idle' | 'connecting' | 'connected'>('idle');
  const [lastMsg, setLastMsg] = useState('');

  useEffect(() => {
    const ws = new SessionWS('ws://localhost:8080/ws/session');
    ws.onMessage((m) => {
      if (m.type === 'pong') setLastMsg('pong @ ' + new Date().toISOString());
    });
    setStatus('connecting');
    ws.connect(() => {
      setStatus('connected');
      ws.send({ type: 'ping' });
    });
  }, []);

  return (
    <div style={{ padding: 24 }}>
      <h1>Web Claude</h1>
      <p>Status: {status}</p>
      <p>Last: {lastMsg}</p>
      <button onClick={() => fetch('/api/ping').then(r => r.text()).then(setLastMsg)}>
        REST ping
      </button>
    </div>
  );
}
```

## 7. 常见问题

### Q1: WS 连接失败（前端 Status 一直是 connecting）

- 检查后端是否启动（端口 8080）；
- 检查 vite 代理是否配了 `/ws`；
- Chrome DevTools → Network → WS 看握手是否成功。

### Q2: Flyway 报 "schema differs"

- JPA 用 `ddl-auto: validate`，确保 Entity 字段与 SQL 完全一致；
- 实在不行临时改成 `ddl-auto: update`（仅开发）。

### Q3: 沙箱构建很慢

- Dockerfile 用了 Playwright 安装 chromium，比较大；
- 第一次构建约 5-10 分钟；
- 后续构建走 docker cache 快很多。

### Q4: 端口冲突

- 8080 / 5173 / 5432 / 6379 / 9000 / 9001 任一被占用都会失败；
- 用 `lsof -i :端口` 检查；
- 修改对应 yaml 端口映射。
