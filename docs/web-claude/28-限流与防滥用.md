## 0. 为什么必须做

10 章只有按 token 限流,真实生产场景会遇到的攻击:

| 攻击 | 后果 |
|------|------|
| 机器人批量注册账号 | 数据库爆 + 邀请奖励被刷 |
| 单 IP 高频扫登录 | 撞库成功 |
| 单用户开 1000 个 session | 沙箱池耗尽,其他用户起不来 |
| 单租户跑 50 个长程任务 | Cron 队列堵死 |
| 竞品爬你的 UI / prompt | 模型被逆向 |
| 公共分享链接被暴力枚举 | 他人 session 数据泄露 |
| 一个用户开多设备绕限流 | 限流形同虚设 |

不做的后果:上线一周成本爆炸 + 服务降级 + 数据泄露。

## 1. 多维度限流模型

```
请求 → 提取身份 → 多维度限流 → 通过/拒绝/降级

维度:
  ├─ IP（防扫描）
  ├─ 用户（防个人滥用）
  ├─ 租户（防企业超额）
  ├─ 会话（防单 session 风暴）
  ├─ 任务（防长程任务积压）
  └─ 设备指纹（防多账号）
```

### 1.1 维度与限流类型

| 维度 | 限制 | 实现 | 拒绝策略 |
|------|------|------|---------|
| IP | 60 req/min | Redis 计数 | 429 + Retry-After |
| IP | 5 注册/hour | Redis 计数 | 429 + 验证码升级 |
| IP | 100 登录失败/hour | Redis 计数 | 封禁 1h |
| 用户 | 1000 tokens/min | Redis 计数 | 429 |
| 用户 | 50 sessions/day | Redis 计数 | 429 |
| 用户 | 5 并发 session | Redis set | 429 |
| 租户 | 100k tokens/min | Redis 计数 | 429 + 告警 |
| 租户 | 50 并发 session | Redis set | 排队 |
| 租户 | 10 并发长程任务 | Redis set | 排队 |
| 设备 | 1 个用户 / 设备 | 设备指纹 | 拒绝注册 |
| 分享链接 | 100 访问/hour | Redis 计数 | 429 |

### 1.2 通用 RateLimiter

```java
// 本代码仅作学习材料参考
@Component
public class RateLimiter {

    private final StringRedisTemplate redis;

    public boolean tryAcquire(String key, int max, Duration window) {
        String k = "rate:" + key;
        Long current = redis.opsForValue().increment(k);
        if (current != null && current == 1L) {
            redis.expire(k, window);
        }
        return current != null && current <= max;
    }

    public boolean tryAcquireConcurrent(String key, int max, Duration ttl) {
        String k = "conc:" + key;
        Long size = redis.opsForSet().size(k);
        if (size != null && size >= max) return false;
        String member = UUID.randomUUID().toString();
        redis.opsForSet().add(k, member);
        redis.expire(k, ttl);  // 防止泄漏
        return true;
    }

    public void releaseConcurrent(String key, String member) {
        redis.opsForSet().remove("conc:" + key, member);
    }
}
```

### 1.3 限流拦截器

```java
// 本代码仅作学习材料参考
@Component
@Order(1)
public class RateLimitFilter extends OncePerRequestFilter {

    private final RateLimiter limiter;

    @Override
    protected void doFilterInternal(HttpServletRequest req, HttpServletResponse resp,
                                    FilterChain chain) throws ServletException, IOException {
        String ip = clientIp(req);
        TenantContext ctx = TenantContext.current();

        // 1. IP 维度（所有请求）
        if (!limiter.tryAcquire("ip:" + ip, 60, Duration.ofMinutes(1))) {
            return reject(resp, 429, "ip rate limit", 60);
        }

        // 2. 用户维度（鉴权后）
        if (ctx != null) {
            if (!limiter.tryAcquire("user:" + ctx.userId(), 200, Duration.ofMinutes(1))) {
                return reject(resp, 429, "user rate limit", 60);
            }
            if (!limiter.tryAcquire("tenant:" + ctx.tenantId(), 1000, Duration.ofMinutes(1))) {
                return reject(resp, 429, "tenant rate limit", 60);
            }
        }

        chain.doFilter(req, resp);
    }

    private String clientIp(HttpServletRequest req) {
        String xff = req.getHeader("X-Forwarded-For");
        if (xff != null) return xff.split(",")[0].trim();
        return req.getRemoteAddr();
    }

    private void reject(HttpServletResponse resp, int status, String msg, long retryAfter) throws IOException {
        resp.setStatus(status);
        resp.setHeader("Retry-After", String.valueOf(retryAfter));
        resp.getWriter().write("{\"error\":\"" + msg + "\"}");
    }
}
```

## 2. 注册与登录防爆破

### 2.1 注册限流

```java
// 本代码仅作学习材料参考
@PostMapping("/api/auth/register")
public ResponseEntity<?> register(@RequestBody RegisterRequest req, HttpServletRequest http) {
    String ip = clientIp(http);

    // 1. IP 维度:每小时 5 次
    if (!limiter.tryAcquire("register:ip:" + ip, 5, Duration.ofHours(1))) {
        return tooMany("每小时注册次数超限");
    }

    // 2. 邮箱维度:每邮箱每小时 1 次
    if (!limiter.tryAcquire("register:email:" + req.email(), 1, Duration.ofHours(1))) {
        return tooMany("该邮箱已注册或刚申请过");
    }

    // 3. 设备指纹:同设备 1 个账号
    String fp = req.deviceFingerprint();
    if (userRepo.existsByDeviceFingerprint(fp)) {
        return badRequest("该设备已注册");
    }

    // 4. 邀请制:校验邀请码
    if (inviteRequired) {
        if (!inviteService.validate(req.inviteCode(), req.email())) {
            return badRequest("邀请码无效");
        }
    }

    // 5. CAPTCHA(可选:第 3 次失败后强制)
    if (failedRegisterCount(ip) >= 3 && !captchaService.verify(req.captcha())) {
        return badRequest("验证码错误");
    }

    // 创建用户(待激活)
    User u = userService.create(req);
    mailService.sendActivation(u);

    return ok(u);
}
```

### 2.2 登录失败锁定

```java
// 本代码仅作学习材料参考
@PostMapping("/api/auth/login")
public ResponseEntity<?> login(@RequestBody LoginRequest req, HttpServletRequest http) {
    String ip = clientIp(http);
    String key = "login:" + ip + ":" + req.email();

    // 1. 同 IP + 邮箱:5 次失败锁 1h
    Long fails = incrementFailure(key, Duration.ofHours(1));
    if (fails > 5) {
        return tooMany("登录失败次数过多,1 小时后再试");
    }

    User u = userService.authenticate(req);
    if (u == null) {
        return unauthorized("邮箱或密码错误");
    }

    // 成功,清除失败计数
    clearFailure(key);

    // 2. 同 IP 100 次失败封 1h(撞库特征)
    if (totalIpFails(ip) > 100) {
        banService.ban(ip, Duration.ofHours(1));
        return tooMany("可疑流量,已封禁");
    }

    return ok(issueToken(u));
}
```

## 3. 会话与任务并发控制

### 3.1 用户并发 session

```java
// 本代码仅作学习材料参考
@PostMapping("/api/sessions")
public SessionEntity createSession(@RequestBody CreateSessionRequest req) {
    TenantContext ctx = TenantContext.current();
    String userKey = "conc:user:" + ctx.userId();
    String tenantKey = "conc:tenant:" + ctx.tenantId();

    // 用户并发 session ≤ 5
    if (!limiter.tryAcquireConcurrent(userKey, 5, Duration.ofHours(2))) {
        throw new RateLimitException("并发会话过多,关闭一些再试");
    }

    // 租户并发 session ≤ 50
    if (!limiter.tryAcquireConcurrent(tenantKey, 50, Duration.ofHours(2))) {
        throw new RateLimitException("租户并发会话过多");
    }

    SessionEntity s = sessionService.create(req);

    // 把 sessionId 入 set 用于清理
    redis.opsForSet().add(userKey, s.getId().toString());
    return s;
}

// session 结束时释放
@EventListener
public void onSessionEnd(SessionEndEvent e) {
    String userKey = "conc:user:" + e.userId();
    String tenantKey = "conc:tenant:" + e.tenantId();
    redis.opsForSet().remove(userKey, e.sessionId().toString());
    redis.opsForSet().remove(tenantKey, e.sessionId().toString());
}
```

### 3.2 长程任务公平调度

11 章 TaskScheduler 用 Scheduler Lock 单线程处理。生产需要公平:

```java
// 本代码仅作学习材料参考
@Component
public class FairTaskScheduler {

    private final StringRedisTemplate redis;

    @Scheduled(fixedRate = 60_000)
    public void tick() {
        if (!acquireGlobalLock()) return;

        try {
            // 1. 每个租户的待跑任务
            int totalSlots = config.getMaxConcurrentTasks();  // 例如 50
            int used = countActive();

            while (used < totalSlots) {
                // 2. 公平:每个租户按 round-robin 拿
                UUID tenantId = pickTenantRoundRobin();
                if (tenantId == null) break;

                // 3. 单租户并发 ≤ 10
                Long tenantActive = countActiveByTenant(tenantId);
                if (tenantActive >= config.getPerTenantMax()) continue;

                // 4. 拿这个租户的最高优先级待跑任务
                TaskEntity t = taskRepo.findNextForTenant(tenantId);
                if (t == null) continue;

                sessionLauncher.launchForTask(t);
                used++;
            }
        } finally {
            releaseGlobalLock();
        }
    }

    private UUID pickTenantRoundRobin() {
        // 按上次调度时间排序,最久没调度的优先
        return redis.opsForZSet().rangeByScore("scheduler:tenant:last", 0, Instant.now().getEpochSecond(), 0, 1)
            .stream().findFirst().map(UUID::fromString).orElse(null);
    }
}
```

### 3.3 沙箱池公平

```java
// 本代码仅作学习材料参考
// 沙箱池空时,排队而不是无脑拒绝
@Component
public class SandboxQueue {

    private final BlockingQueue<SandboxRequest> queue = new LinkedBlockingQueue<>();

    public CompletableFuture<Sandbox> request(SandboxRequest req) {
        CompletableFuture<Sandbox> future = new CompletableFuture<>();
        req.setFuture(future);
        queue.offer(req);
        return future.orTimeout(120, TimeUnit.SECONDS);
    }

    @Scheduled(fixedRate = 1_000)
    public void drain() {
        while (!pool.isEmpty()) {
            SandboxRequest req = queue.peek();
            if (req == null) break;

            // 公平:同租户不能霸占
            long tenantWaiting = countTenantInQueue(req.tenantId());
            long tenantServed = countTenantServedLast10Min(req.tenantId());
            if (tenantServed / (double) totalServedLast10Min() > tenantWaiting / (double) queue.size() + 0.1) {
                // 该租户已超过公平份额,让位
                queue.offer(queue.poll());
                continue;
            }

            Sandbox s = pool.checkout(Duration.ofSeconds(10));
            if (s == null) break;
            queue.poll().getFuture().complete(s);
        }
    }
}
```

## 4. 公共分享链接防遍历

23 章分享 token 是 20 字节随机(熵 160bit),理论不可枚举。但仍需:

### 4.1 访问限流

```java
// 本代码仅作学习材料参考
@GetMapping("/api/share/{token}")
public ShareView view(@PathVariable String token, HttpServletRequest http) {
    String ip = clientIp(http);

    // 1. 单 IP 单链接 100 次/hour
    if (!limiter.tryAcquire("share:" + ip + ":" + token, 100, Duration.ofHours(1))) {
        throw new RateLimitException("访问过于频繁");
    }

    // 2. 单 IP 全局 1000 次/hour(防遍历)
    if (!limiter.tryAcquire("share-global:" + ip, 1000, Duration.ofHours(1))) {
        throw new RateLimitException("访问过多");
    }

    // 3. 链接不存在次数监控
    Optional<SessionShareLink> opt = shareRepo.findByToken(token);
    if (opt.isEmpty()) {
        Long misses = limiter.increment("share-miss:" + ip, Duration.ofMinutes(10));
        if (misses > 50) {
            // 可疑爆破行为,封 1h
            banService.ban(ip, Duration.ofHours(1));
        }
        throw new NotFoundException("链接无效");
    }

    return buildShareView(opt.get());
}
```

## 5. 防爬虫

### 5.1 识别爬虫特征

```java
// 本代码仅作学习材料参考
@Component
public class BotDetector {

    private static final List<Pattern> BOT_AGENTS = List.of(
        Pattern.compile("(?i)bot|crawler|spider|scrapy|curl|wget|python-requests"),
        Pattern.compile("(?i)headlesschrome"),
        Pattern.compile("(?i)phantomjs|selenium")
    );

    public boolean isBot(HttpServletRequest req) {
        String ua = req.getHeader("User-Agent");
        if (ua == null || ua.length() < 20) return true;
        return BOT_AGENTS.stream().anyMatch(p -> p.matcher(ua).find());
    }
}
```

### 5.2 拦截策略

- 公开页面(首页 / 文档)放行爬虫(有利 SEO);
- API 端点禁止爬虫;
- 高频访问的 IP 加 CAPTCHA challenge;
- 同一 IP 在短时间内访问大量不同 session → 告警;
- WAF(Cloudflare / AWS WAF)前置。

### 6.1 限流响应规范

```java
// 本代码仅作学习材料参考
public class RateLimitException extends RuntimeException {
    private final long retryAfter;
    private final String dimension;  // ip / user / tenant
    private final int limit;

    public ResponseEntity<ErrorResponse> toResponse() {
        return ResponseEntity.status(429)
            .header("Retry-After", String.valueOf(retryAfter))
            .header("X-RateLimit-Limit", String.valueOf(limit))
            .header("X-RateLimit-Dimension", dimension)
            .body(new ErrorResponse("rate_limited", "..."));
    }
}
```

```ts
  if (err.response?.status === 429) {
```

## 11. 对照:与"10 章仅 token 限流"的差异

| 维度 | 10 章 | 本章 |
|------|-------|------|
| 维度 | token / min | IP / 用户 / 租户 / 会话 / 任务 / 设备 |
| 注册爆破 | 无 | §2 CAPTCHA + 邀请制 |
| 公平调度 | 无 | §3 沙箱 round-robin |
| 分享链接 | 无 | §4 防 32 字节 token 遍历 |
| 防爬虫 | 无 | §5 指纹 + 行为检测 |
| 降级 | 拒绝 | §6 降模型不拒绝 |
| 监控 | token 用量 | §7 多维度 + 告警 |

**结论**:10 章是"额度",本章是"防滥用 + 公平",成本治理的硬约束。

## 12. 避坑:限流与防滥用常踩的雷

| 雷 | 现象 | 规避 |
|----|------|------|
| 滑动窗口开销 | 每请求扫 ZSET | 用令牌桶 + 采样 |
| 多设备绕过 | 用户开手机+电脑双倍 | §1 设备指纹聚合用户 |
| 降级误伤 | 高频付费用户被降模型 | 付费 plan 跳过降级 |
| 限流风暴 | 全局限流触发后服务雪崩 | §6 渐进降级 + 熔断 |
| 公平调度饿死 | 小租户一直排队 | §3 weighted fair queueing |
| CAPTCHA 拦正常用户 | 视障用户被卡 | audio CAPTCHA + 白名单 |
| 设备指纹误判 | 同公司同 IP 误同设备 | 多维度聚合 + 阈值 |
| 邀请制漏 | 假邮箱注册 | email verify + domain 白名单 |
| Redis 限流单点 | Redis 挂了限流失效 | 本地兜底 + 多副本 |
| 链接遍历残留 | 旧 token 仍有效 | §4 强制 expire + 撤销列表 |
