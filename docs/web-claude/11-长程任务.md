# 11 - 长程任务：feature_list + Cron + 自动续跑

> 本章目标：实现 Anthropic 推荐的"Initializer + Coding Agent"长程任务范式。
> 完成后：用户提一个"做个 todo app"需求，平台能自动拆成多个 feature，跨多个 session 自动推进。

> - 跨 session 上下文交接：[14 章](./14-上下文工程.md) §10 summary.json；

## 1. 痛点：Agent 一旦超出单 session 就失控

到这里你的 Agent 已经能在单 session 内做很多事——读文件、写代码、跑命令。但只要任务**稍微复杂一点**（比如"做个 todo app"），就会卡在三种死法之一：

1. **context 爆掉**：单 session 干到一半 token 超限，autocompact 后上下文丢失，agent 忘了在干嘛
2. **崩溃无救**：服务重启 / 浏览器关 / 模型限流 → 任务直接断，没法续接
3. **进度不可见**：用户问"现在到哪了"——agent 答不上来，因为它"以为自己还差 5 个 feature"

> 这是 Agent 产品从 "demo" 走向 "能用" 的最大门槛。Anthropic 在 *Effective Harnesses* 一文中给出的范式：**Initializer Agent 拆任务 → Coding Agent 一 session 一 feature → Cron 拉新 session 续跑 → 进度外部化到文件**。
>
> 这一章是这个范式的最简实现。**13 章 Harness 工程**是它的完整版（加 guardrails / protocol / multi-agent），但你不读完 11 章直接看 13 会一头雾水。

## 2. 设计要点（来自调研笔记 §3.3）

- Initializer Agent：生成 feature_list.json / claude-progress.txt / init.sh；
- Coding Agent：每个 session 选最高优先级未完成 feature；
- 状态外部化到 S3；
- Cron + Scheduler Lock 触发新 session。

```java
// 本代码仅作学习材料参考
package org.demo02.webclaude.task;

public class InitializerPrompt {

    public static String build(String requirement) {
        return """
            你是一个长程任务的 Initializer Agent。
            
            用户需求：%s
            
            请完成以下工作：
            
            1. 创建 `feature_list.json`，包含 200+ 个 feature，覆盖完整应用所需的所有功能。
               每个 feature 包含字段：
                 - id: F001, F002, ...
                 - title: 简短描述
                 - priority: 1 (最高) 到 5 (最低)
                 - passes: 一律 false
                 - test_cmd: 验证命令
                 - notes: 注意事项
            
            2. 创建 `claude-progress.txt`，初始只写一行：
               [initialized] 任务已初始化
            
            3. 创建 `init.sh`，启动开发服务器 + 跑基础 e2e 测试。
            
            4. 做第一个 git commit："init: scaffold project"。
            
            强约束：
              - 不许删除/修改任何测试文件；
              - 不许把测试断言改成空操作；
              - feature_list.json 必须是 JSON 格式（不是 Markdown）；
              - 一开始所有 passes=false。
            """.formatted(requirement);
    }
}
```

```java
// 本代码仅作学习材料参考
package org.demo02.webclaude.task;

public class CodingAgentPrompt {

    public static String build() {
        return """
            你是一个长程任务的 Coding Agent，负责在单个 session 内推进任务。
            
            请严格按以下步骤：
            
            1. `pwd` 确认工作目录。
            2. 读 `git log --oneline -20` 和 `claude-progress.txt` 了解之前进展。
            3. 读 `feature_list.json`，找出 priority 最高且 passes=false 的 feature。
            4. 跑 `bash init.sh` 确认基线工作。
            5. 实现这个 feature。
            6. 跑 test_cmd 验证通过。
            7. 更新 feature_list.json，把这个 feature 的 passes 改成 true。
            8. 追加 claude-progress.txt：
               [完成][F0XX] 一句话总结
            9. git commit -m "feat(F0XX): 完成什么功能"
            10. 写一段 summary（给下一次 session 看）。
            
            强约束：
              - 一个 session 只做一个 feature；
              - 不许删/改测试；
              - 不许同时改多个 feature；
              - 测试不通过不更新 passes；
              - 必须用 Playwright 测试 UI 类 feature。
            """;
    }
}
```

```java
// 本代码仅作学习材料参考
package org.demo02.webclaude.task;

import org.springframework.stereotype.Service;
import software.amazon.awssdk.core.sync.RequestBody;
import software.amazon.awssdk.services.s3.S3Client;
import software.amazon.awssdk.services.s3.model.PutObjectRequest;

import java.nio.charset.StandardCharsets;
import java.util.UUID;

@Service
public class TaskService {

    private final S3Client s3;
    private final TaskRepository repo;

    public String storeFeatureList(UUID taskId, String json) {
        String key = "tasks/" + taskId + "/feature_list.json";
        s3.putObject(PutObjectRequest.builder().bucket("web-claude").key(key).build(),
            RequestBody.fromString(json, StandardCharsets.UTF_8));
        return key;
    }

    public String storeProgress(UUID taskId, String content) {
        String key = "tasks/" + taskId + "/claude-progress.txt";
        s3.putObject(PutObjectRequest.builder().bucket("web-claude").key(key).build(),
            RequestBody.fromString(content, StandardCharsets.UTF_8));
        return key;
    }

    public void appendProgress(UUID taskId, String line) {
        String existing = loadProgress(taskId);
        storeProgress(taskId, existing + "\n" + line);
    }

    public String loadProgress(UUID taskId) {
        // S3 get，省略
        return "";
    }
}
```

## 7. Scheduler Lock + Cron

新建 `src/main/java/org/demo02/webclaude/task/TaskScheduler.java`：

```java
// 本代码仅作学习材料参考
package org.demo02.webclaude.task;

import org.springframework.dao.DataIntegrityViolationException;
import org.springframework.data.redis.core.StringRedisTemplate;
import org.springframework.scheduling.annotation.Scheduled;
import org.springframework.stereotype.Component;

import java.time.Duration;
import java.time.Instant;
import java.util.List;
import java.util.UUID;

@Component
public class TaskScheduler {

    private final StringRedisTemplate redis;
    private final TaskRepository taskRepo;
    private final TaskService taskService;
    private final SessionLauncher sessionLauncher;

    private static final String LOCK_KEY = "scheduler:lock";
    private static final Duration LOCK_TTL = Duration.ofSeconds(30);

    @Scheduled(fixedRate = 60_000)
    public void tick() {
        String lockToken = UUID.randomUUID().toString();
        Boolean acquired = redis.opsForValue()
            .setIfAbsent(LOCK_KEY, lockToken, LOCK_TTL);
        if (!Boolean.TRUE.equals(acquired)) return;

        try {
            List<TaskEntity> candidates = taskRepo.findSchedulable(Instant.now().minusSeconds(120));
            for (TaskEntity t : candidates) {
                try {
                    sessionLauncher.launchForTask(t);
                } catch (Exception e) {
                    t.setStatus("failed");
                    t.setErrorMsg(e.getMessage());
                    taskRepo.save(t);
                }
            }
        } finally {
            // Lua 释放锁（避免误删别人的锁）
            String current = redis.opsForValue().get(LOCK_KEY);
            if (lockToken.equals(current)) {
                redis.delete(LOCK_KEY);
            }
        }
    }
}
```

```java
// 本代码仅作学习材料参考
@Component
public class SessionLauncher {

    public SessionEntity launchForTask(TaskEntity task) {
        // 1. 创建新 session
        SessionEntity s = newSession(task);
        // 2. 拉起沙箱
        sandboxManager.startForTask(s.getId(), task.getId());
        // 3. 注入 system prompt（Coding Agent prompt）
        s.setSystemPrompt(CodingAgentPrompt.build());
        // 4. 异步起 Agent Loop
        agentLoop.run(s.getId(), List.of(), "开始", null, ...).subscribe();
        return s;
    }

    public SessionEntity launchInitializer(TaskEntity task) {
        SessionEntity s = newSession(task);
        sandboxManager.start(s.getId());
        s.setSystemPrompt(InitializerPrompt.build(task.getRequirement()));
        agentLoop.run(s.getId(), List.of(), "开始", null, ...).subscribe();
        return s;
    }
}
```

## 9. 完整任务流

### 7.1 创建任务

```java
// 本代码仅作学习材料参考
@PostMapping("/api/tasks")
public TaskEntity create(@RequestBody CreateTaskRequest req) {
    TaskEntity task = new TaskEntity();
    task.setId(UUID.randomUUID());
    task.setTenantId(currentTenant());
    task.setUserId(currentUser());
    task.setTitle(req.title());
    task.setRequirement(req.requirement());
    task.setStatus("initializing");
    taskRepo.save(task);

    // 立即拉起 Initializer
    sessionLauncher.launchInitializer(task);

    return task;
}
```

### 7.2 Initializer 完成后

Agent Loop 完成时，检测到产出了 `feature_list.json`：

```java
// 本代码仅作学习材料参考
if (initialRun && filesWritten.contains("feature_list.json")) {
    String key = taskService.storeFeatureList(taskId, content);
    task.setFeatureListKey(key);
    task.setStatus("running");
    task.setStartedAt(Instant.now());
    taskRepo.save(task);
```

### 7.3 Cron 触发 Coding Agent

每分钟扫描：
- status = running；
- 当前无 active session；
- 上一个 session 结束 ≥ 2 分钟。

满足 → 起新 session → 注入 Coding Agent prompt。

### 7.4 任务完成


新建 `src/components/TaskTimeline.tsx`：

```tsx
// 本代码仅作学习材料参考
export default function TaskTimeline({ task }: { task: TaskWire }) {
  return (
    <div style={{ padding: 24 }}>
      <h2>{task.title}</h2>
      <div>状态：{task.status}</div>
      <FeatureList features={task.features} />
      <ProgressLog url={task.progressUrl} />
      <SessionList sessions={task.sessions} />
    </div>
  );
}

function FeatureList({ features }: { features: Feature[] }) {
  return (
    <div>
      <h3>功能列表（{features.filter(f => f.passes).length}/{features.length}）</h3>
      <ul>
        {features.map(f => (
          <li key={f.id} style={{ color: f.passes ? 'green' : 'gray' }}>
            [{f.id}] {f.title} {f.passes ? '✓' : '…'}
          </li>
        ))}
      </ul>
    </div>
  );
}

function ProgressLog({ url }: { url: string }) {
  const [text, setText] = useState('');
  useEffect(() => {
    const id = setInterval(() =>
      fetch(url).then(r => r.text()).then(setText), 5000);
    return () => clearInterval(id);
  }, [url]);
  return <pre>{text}</pre>;
}
```

## 12. 对照：与"单 session 一次性搞定"的能力差异

| 维度 | 单 session | 长程任务（11 章） |
|------|-----------|------------------|
| 任务时长 | 单 turn / 几分钟 | ✅ 数小时～数天 |
| 跨 session | ❌ | ✅ |
| 崩溃恢复 | ❌ | ✅ Cron 自动起新 session |
| 进度可见 | ❌ | ✅ claude-progress.txt |
| 任务拆解 | Agent 自己理解 | ✅ Initializer 显式拆 |
| 并行加速 | ❌ | ⚠️（15 章 Subagent 才并行）|

## 13. 避坑：长程任务常踩的雷

| 雷 | 现象 | 规避 |
|----|------|------|
| Scheduler Lock 失效，多实例同时拉 session | 同一 feature 被跑两遍 | Redis 分布式锁 + lease timeout |
| Cron 周期太短 | 上一个还没完，下一个又起 | 锁未释放时跳过 |
| feature 顺序依赖被忽略 | feature-2 假设 feature-1 已完成 | feature_list 显式声明依赖 |
| progress.txt 写入竞态 | 多 session 同时改写 | 单 writer 模式 / 乐观锁 |
| Coding Agent 跨 feature 改 | 一个 session 改了多个 feature | system prompt 强约束 + post-check |
| Initializer 拆得太粗 | 每个 feature 还是要好几 session | 拆到"一个 session 可完成"粒度 |
| Agent 删测试作弊 | feature 看起来过了实际没过 | 13 章 TestVerifier |
| 进度文件丢失 | S3 故障任务断 | 18 章 §12 长程任务特殊恢复 |
| Cron 死锁 | 锁过期前实例挂了 | lease timeout + 心跳续约 |
| 无限循环 | feature 永远 in_progress | max retry + 强制失败 |

## 14. 关键细节

### 10.1 状态外部化

所有跨 session 的状态都在 S3（feature_list / progress）和 DB（task / session / messages），不依赖 LLM 记忆。

### 10.2 中断恢复

- 关闭浏览器：session 持续跑（直到 end_turn）；
- 重启服务：Cron 下次 tick 时拉起新 session；
- 工作区：通过 git 状态恢复（每次 session 都 commit）。

### 10.3 失败处理

- 单个 feature 失败：标 passes=false，写 progress 说明原因，继续下一个；
- 多个 session 连续失败：暂停任务，等待用户介入。
