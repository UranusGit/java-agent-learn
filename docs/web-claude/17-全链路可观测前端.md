## 1. 设计哲学

### 0.1 三档透明度

| 模式 | 看到什么 | 适用人群 |
|------|---------|---------|
| 简洁 | user / assistant text / artifact | 普通用户 |
| **详细（默认）** | + thought / tool_call / tool_result / decision | 工程师 |
| 调试 | + 完整 prompt / model_request / mcp / 错误堆栈 | 排查 |

用户随时切换。

### 0.2 事件粒度原则

- **原子性**：一个事件 = 一个可渲染、可审计的原子操作；
- **完整性**：每个事件独立可用（不依赖前后事件的 state）；
- **可重放**：事件按 seq 单调递增，重连按 seq 重放；
- **可折叠**：同类事件 UI 上能批量展示。

### 0.3 类比

| 业界产品 | 透明度 |
|---------|--------|
| ChatGPT | 简洁（只看到 final answer）|
| Claude.ai | 简洁 + thinking |
| Claude Code CLI | 详细（流式 + 工具）|
| **web-claude（本章）** | **三档可切换，最详细超 CC** |
| Devin / Factory | 详细 + 拓扑图 |

## 2. 12 类原子事件

### 1.1 完整清单

```
1.  user_input           用户发消息
2.  thought              Agent 思考（reasoning / planning / analyzing）
3.  tool_call            Agent 调工具
4.  tool_result          工具返回
5.  mcp_call             MCP 工具调用
6.  mcp_result           MCP 工具返回
7.  mcp_server_status    MCP server 状态变化
8.  model_request        发给模型的完整请求
9.  model_stream         模型流式输出
10. state                Agent 状态变化（thinking/streaming/end_turn）
11. decision             HITL 决策（requested/made）
12. system               压缩/fallback/错误/恢复/任务阶段
```

### 1.2 统一 Envelope

```typescript
// 本代码仅作学习材料参考
export interface AgentEvent {
  type:
    | 'user_input' | 'thought' | 'tool_call' | 'tool_result'
    | 'mcp_call' | 'mcp_result' | 'mcp_server_status'
    | 'model_request' | 'model_stream'
    | 'state' | 'decision' | 'system';

  seq: number;                    // 单调递增
  sessionId: string;
  taskId?: string;
  subagentId?: string;
  at: string;                     // ISO timestamp
  level: 'info' | 'debug' | 'critical';
  data: EventData;                // type-specific
}
```

### 3.2 EventBus

```java
// 本代码仅作学习材料参考
@Component
public class AgentEventBus {

    private final Sinks.Many<AgentEvent> sink = Sinks.many().multicast().onBackpressureBuffer();
    private final AgentEventRepository repo;
    private final SessionSeqAllocator seqAllocator;

    public Mono<Void> emit(UUID sessionId, String type, String level, Object data) {
        long seq = seqAllocator.next(sessionId);
        AgentEvent e = new AgentEvent(UUID.randomUUID(), sessionId, seq, type, level, data, Instant.now());
        return repo.save(e).then(Mono.fromRunnable(() -> sink.tryEmitNext(e))).then();
    }

    public Flux<AgentEvent> subscribe(UUID sessionId) {
        return sink.asFlux().filter(e -> e.sessionId().equals(sessionId));
    }
}
```

## 5. Agent Loop 埋点策略

Agent Loop 的每个原子操作都要 emit 事件。改造 04 章的 AgentLoopV2：

```java
// 本代码仅作学习材料参考
public Flux<State> run(UUID sessionId, List<Message> history, String userInput,
                       AbortHandle abort, ToolRegistry tools,
                       PermissionMiddleware perm, ToolContext ctx,
                       AgentEventBus bus) {
    return Flux.create(sink -> {
        bus.emit(sessionId, "user_input", "info",
            Map.of("text", userInput)).subscribe();

        runLoopWithEvents(sessionId, history, userInput, sink, abort, tools, perm, ctx, bus);
    });
}

private void runLoopWithEvents(...) {
    bus.emit(sessionId, "thought", "info",
        Map.of("subtype", "planning", "text", "开始处理用户请求", "modelGenerated", false))
        .subscribe();

    // 装配 prompt（接 14 章）
    String system = systemPromptAssembler.assemble(ctx, session, task);
    List<Message> compacted = compact(history, modelLimit);

    // emit model_request
    bus.emit(sessionId, "model_request", "debug",
        Map.of(
            "requestId", reqId,
            "model", model,
            "systemPrompt", Map.of(
                "length", system.length(),
                "preview", system.substring(0, Math.min(100, system.length())),
                "fullUrl", "/api/sessions/" + sessionId + "/requests/" + reqId + "/system"
            ),
            "messages", compacted.stream().map(m -> Map.of(
                "role", m.role(),
                "preview", preview(m),
                "fullUrl", "/api/sessions/" + sessionId + "/requests/" + reqId + "/messages/" + m.id()
            )).toList(),
            "toolsAvailable", tools.schemasForMode(ctx.mode()).stream().map(t -> t.get("name")).toList(),
            "tokensIn", estimator.estimateAll(compacted)
        )).subscribe();

    bus.emit(sessionId, "model_stream", "info",
        Map.of("phase", "start", "requestId", reqId)).subscribe();

    chatClient.prompt().messages(...).stream().chatResponse()
        .doOnNext(chunk -> {
            String delta = chunk.getResult().getOutput().getText();
            if (delta != null) {
                bus.emit(sessionId, "model_stream", "info",
                    Map.of("phase", "delta", "requestId", reqId, "text", delta)).subscribe();
            }
        })
        .doOnComplete(() -> {
            bus.emit(sessionId, "model_stream", "info",
                Map.of("phase", "done", "requestId", reqId)).subscribe();

            // 检测 tool_use
            for (ContentBlock.ToolUse tu : extractToolUses(lastResponse)) {
                handleToolCall(tu, sessionId, sink, bus, ...);
            }
        })
        .subscribe();
}

private void handleToolCall(ContentBlock.ToolUse tu, UUID sessionId, ..., AgentEventBus bus) {
    bus.emit(sessionId, "tool_call", "info",
        Map.of(
            "toolCallId", tu.id(),
            "tool", Map.of(
                "kind", tu.isMcp() ? "mcp" : "builtin",
                "name", tu.name(),
                "mcpServer", tu.mcpServer()
            ),
            "input", tu.input(),
            "inputPreview", previewInput(tu)
        )).subscribe();

    // 权限
    Decision d = perm.evaluate(...).join();
    bus.emit(sessionId, "decision", "info",
        Map.of("phase", "requested", "decisionId", ..., "kind", "tool_approval", ...)).subscribe();

    if (d.action() == Action.ASK) {
        // 挂起，等用户答（详见 19/20 章）
        return;
    }

    // 执行
    ToolResult r;
    if (tu.isMcp()) {
        bus.emit(sessionId, "mcp_call", "info", Map.of(...)).subscribe();
        r = mcpClientManager.callTool(...);
        bus.emit(sessionId, "mcp_result", "info",
            Map.of("toolCallId", tu.id(), "durationMs", ..., "content", r.content())).subscribe();
    } else {
        long start = System.currentTimeMillis();
        r = toolRegistry.get(tu.name()).apply(tu.input(), ctx).join();
        bus.emit(sessionId, "tool_result", "info",
            Map.of(
                "toolCallId", tu.id(),
                "isError", r.isError(),
                "stdout", r.content(),
                "isTruncated", r.isTruncated(),
                "durationMs", System.currentTimeMillis() - start
            )).subscribe();
    }

    if (r.isArtifact()) {
        bus.emit(sessionId, "artifact", "info", Map.of(...)).subscribe();
    }
}
```

## 9. 避坑：全链路可观测常踩的雷

| 雷 | 现象 | 规避 |
|----|------|------|
| 事件风暴（一秒上百条） | 前端卡死 / WS 拥塞 | 后端聚合 + 前端节流 |
| tool_result 大字段直推 | WS 单帧 10MB 卡死 | §6 大字段按需加载 |
| React 列表不虚拟化 | 1000 条活动卡 | react-window / react-virtualized |
| 全部事件都展开渲染 | DOM 节点爆炸 | 默认折叠 + 按需展开 |
| event_id 不单调 | 重连重放乱序 | 严格单调 + 前端按序插入 |
| 重复事件去重失败 | 同一事件显示两遍 | event_id 幂等去重 |
| subagent 拓扑爆炸 | 50 个子 agent 一张图画不出 | 默认折叠 + 仅显示活跃子树 |
| 事件 schema 漂移 | 老前端不识别新事件 | schema 版本 + 兼容字段 |
| trace_id 没透传到前端 | Jaeger 看不到对应前端行为 | 事件携带 trace_id + 前端 Sentry 关联 |
| 事件 DB 撑爆 | 几个月几亿行 | 分区表 + 冷数据归档 |

### 9.3 debug（调试）

- `model_request`（完整）

```typescript
// 本代码仅作学习材料参考
function filterByLevel(events: AgentEvent[], level: string): AgentEvent[] {
    const allowlist: Record<string, string[]> = {
        concise: ['user_input', 'model_stream', 'artifact', 'decision'],
        detailed: ['user_input', 'thought', 'model_stream', 'tool_call', 'tool_result',
                   'mcp_call', 'mcp_result', 'state', 'decision', 'artifact'],
        debug: ['*'],  // 全部
    };
    const allowed = allowlist[level];
    return events.filter(e => allowed.includes('*') || allowed.includes(e.type));
}
```
