# 25 - 记忆与个性化系统：跨 session 的用户偏好与项目约定

> 本章目标：让 agent 跨 session 记住"你是谁、你偏好什么、这个项目什么约定"，避免每次重问。
> 完成后：用户在同一项目下开新 session，agent 自动知道用 React / TypeScript / 不写注释 / commit 信息中文；用户偏好（活动流显示级别 / 主题）也跨设备同步。
>
> **关联章节**：
> - System Prompt 装配：[14 章 SystemPromptAssembler](./14-上下文工程.md)（本章是其中"用户层"+"项目层"的来源）；
> - 跨设备偏好同步：[22 章 §5](./22-跨标签页与实时协作.md)；
> - 记忆中的敏感字段：[24 章 SecretGuard](./24-智能体安全.md)；
> - 工具：SaveMemory / ListMemories / DeleteMemory。

---

## 0. 为什么需要记忆系统

Claude Code CLI 用 `CLAUDE.md` 项目根文件保存项目约定，web-claude 没有本地文件系统，需要数据库 + UI。

没有记忆系统的痛苦：

```
session 1: 用户："我们用 React + TypeScript"
session 2: 用户："做个 todo"
agent:  "用什么技术栈？"  # ← 又问一遍
session 3: 用户："这里不写注释"
session 4: agent: "我加了详细注释..."  # ← 又错
```

有记忆系统：

```
session 1: 用户："我们用 React + TypeScript"
           → agent 自动调用 SaveMemory("tech_stack", "React + TS")
session 2: 用户："做个 todo"
           → system prompt 注入：tech_stack=React+TS
           → agent 不问技术栈，直接做
```

## 1. 记忆分类

| 类型 | 范围 | 内容 | 写入者 |
|------|------|------|--------|
| **user_global** | 用户级，跨所有项目 | 显示偏好 / 主题 / 语言 / 个人习惯 | 用户 / agent |
| **user_project** | 用户 × 项目 | "在这个项目我用 React" | 用户 / agent |
| **project_shared** | 项目级共享（等同 CLAUDE.md） | 项目约定 / 工作流 / 命名风格 | 项目管理员 |
| **session_episodic** | 单 session 内 | 临时决策、近期上下文 | agent |
| **task_lessons** | 任务级 | "上次跑这个 task 失败原因 X" | agent |

## 2. 表结构

```sql
-- 本代码仅作学习材料参考
CREATE TABLE memories (
    id UUID PRIMARY KEY,
    tenant_id UUID NOT NULL,
    user_id UUID,                   -- null = project_shared
    project_id UUID,                -- null = user_global
    scope VARCHAR(32) NOT NULL,     -- user_global / user_project / project_shared / session_episodic / task_lessons
    category VARCHAR(64) NOT NULL,  -- tech_stack / naming / workflow / preference / lesson / ...
    key VARCHAR(255) NOT NULL,
    value TEXT NOT NULL,
    confidence FLOAT NOT NULL DEFAULT 1.0,  -- 0-1，agent 推断的信心
    source VARCHAR(32) NOT NULL,    -- user_explicit / agent_inferred / imported_claude_md
    expires_at TIMESTAMP,           -- 可过期
    created_at TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMP NOT NULL DEFAULT NOW(),
    UNIQUE (tenant_id, user_id, project_id, scope, key)
);
CREATE INDEX idx_memories_lookup ON memories(tenant_id, user_id, project_id, scope);

-- 嵌入索引（向量检索用，可选 v2）
CREATE INDEX idx_memories_embedding ON memories USING ivfflat (embedding vector(1536>));
```

> `embedding` 字段 v2 用，v1 先用 SQL LIKE + key 命名约定。

## 3. 记忆写入

### 3.1 工具：SaveMemory

```java
// 本代码仅作学习材料参考
@Component
public class SaveMemoryTool implements Tool {

    private final MemoryService memoryService;

    @Override public String name() { return "SaveMemory"; }

    @Override public Map<String, Object> schema() {
        return Map.of(
            "type", "object",
            "properties", Map.of(
                "scope", Map.of("type", "string", "enum",
                    List.of("user_global", "user_project", "project_shared")),
                "category", Map.of("type", "string"),
                "key", Map.of("type", "string", "description", "短标识符，如 tech_stack"),
                "value", Map.of("type", "string"),
                "confidence", Map.of("type", "number", "default", 0.8)
            ),
            "required", List.of("scope", "category", "key", "value")
        );
    }

    @Override
    public CompletableFuture<ToolResult> apply(Map<String, Object> input, ToolContext ctx) {
        String scope = (String) input.get("scope");

        // 权限检查：project_shared 需要项目管理员
        if ("project_shared".equals(scope) && !ctx.isProjectAdmin()) {
            return CompletableFuture.completedFuture(ToolResult.error("仅项目管理员可写 project_shared 记忆"));
        }

        Memory m = new Memory(
            UUID.randomUUID(),
            ctx.tenantId(),
            "user_global".equals(scope) ? ctx.userId() : ctx.userId(),
            "user_global".equals(scope) ? null : ctx.projectId(),
            scope,
            (String) input.get("category"),
            (String) input.get("key"),
            secretGuard.scan((String) input.get("value")),  // 24 章扫描
            ((Number) input.getOrDefault("confidence", 0.8)).doubleValue(),
            "agent_inferred",
            null
        );

        memoryService.upsert(m);
        return CompletableFuture.completedFuture(ToolResult.text("saved: " + input.get("key")));
    }
}
```

### 3.3 教 agent 何时写

System prompt 追加：

```
# When to Save Memory

Save a memory when you observe durable information that will help future sessions:
- The user states a preference ("用 React", "不要写注释", "commit 用中文")
- The user corrects your behavior ("不是这样，应该 X")
- You discover project conventions from existing code
- You make an important decision that future sessions should respect

Do NOT save:
- Ephemeral task state (use task progress files instead)
- Secrets / credentials (NEVER)
- Single-turn context (already in messages)

Default scope = user_project. Use project_shared for team conventions
verified by the project admin.
```

### 3.4 显式 vs 推断

| 来源 | 写入流程 |
|------|---------|
| `user_explicit` | 用户在记忆管理 UI 手动添加 |
| `agent_inferred` | agent 调 SaveMemory 工具，需用户确认（avoid 错记） |
| `imported_claude_md` | 从用户的 CLAUDE.md 导入（一次性迁移） |

避免 agent 错记：

## 4. 记忆读取与注入

### 4.1 SystemPromptAssembler 接入

14 章的 SystemPromptAssembler 是 4 层（平台 / 租户 / 项目 / 会话），本章是其中"项目层"和"会话层"的部分：

```java
// 本代码仅作学习材料参考
@Component
public class MemoryPromptSection implements PromptSection {

    private final MemoryService memoryService;

    @Override
    public String render(PromptContext ctx) {
        List<Memory> all = memoryService.loadRelevant(
            ctx.tenantId(), ctx.userId(), ctx.projectId(), ctx.sessionId()
        );

        if (all.isEmpty()) return "";

        StringBuilder sb = new StringBuilder("\n# Memory (User & Project Context)\n\n");

        // 按 category 分组
        Map<String, List<Memory>> grouped = all.stream()
            .collect(Collectors.groupingBy(Memory::getCategory));

        grouped.forEach((cat, items) -> {
            sb.append("## ").append(cat).append("\n");
            items.forEach(m -> sb.append("- ").append(m.getKey())
                .append(": ").append(m.getValue())
                .append(" (").append(m.getScope()).append(")\n"));
            sb.append("\n");
        });

        sb.append("""
            以上信息由 SaveMemory 工具记录，是跨 session 持久化的。
            如果与用户当前指令冲突，以用户当前指令为准。
            """);

        return sb.toString();
    }
}
```

### 4.2 loadRelevant 检索策略

```java
// 本代码仅作学习材料参考
public List<Memory> loadRelevant(UUID tenantId, UUID userId, UUID projectId, UUID sessionId) {
    // 1. user_global（与 project 无关的偏好）
    List<Memory> global = repo.findByScope(tenantId, userId, null, "user_global");

    // 2. user_project（用户在该项目的偏好）
    List<Memory> userProj = projectId == null ? List.of() :
        repo.findByScope(tenantId, userId, projectId, "user_project");

    // 3. project_shared（项目共享约定，等同 CLAUDE.md）
    List<Memory> shared = projectId == null ? List.of() :
        repo.findProjectShared(tenantId, projectId);

    // 4. session_episodic（临时）
    List<Memory> episodic = sessionId == null ? List.of() :
        repo.findBySession(sessionId);

    // 5. task_lessons（最近的）
    List<Memory> lessons = repo.findRecentLessons(tenantId, userId, 10);

    return combine(global, userProj, shared, episodic, lessons);
}
```

### 4.3 Token 预算

记忆可能爆炸（用户存了 100 条），需控制总 token：

```java
// 本代码仅作学习材料参考
private List<Memory> trimByTokenBudget(List<Memory> all, int budget) {
    int used = 0;
    List<Memory> kept = new ArrayList<>();
    // 优先级：shared > user_project > user_global > lessons > episodic
    for (Memory m : sortByPriority(all)) {
        int est = tokenEstimator.estimate(m.getValue());
        if (used + est > budget) continue;
        kept.add(m);
        used += est;
    }
    return kept;
}
```

默认预算：`shared: 2K`、`user: 1K`、`episodic: 500`。

## 7. 记忆向量检索（v2）

当记忆条数 > 100 时，SQL LIKE 不够用，加 embedding：

```java
// 本代码仅作学习材料参考
@Component
public class MemoryEmbeddingService {

    private final EmbeddingModel embedder;  // Spring AI

    public void onMemorySaved(Memory m) {
        float[] vec = embedder.embed(m.getKey() + ": " + m.getValue());
        repo.updateEmbedding(m.getId(), vec);
    }

    public List<Memory> searchByQuery(UUID tenantId, UUID userId, UUID projectId, String query, int topK) {
        float[] q = embedder.embed(query);
        return repo.searchByVector(tenantId, userId, projectId, q, topK);
    }
}
```

`loadRelevant` 升级：

```java
// 本代码仅作学习材料参考
public List<Memory> loadRelevant(...) {
    // 基础：scope 检索
    List<Memory> base = loadByScope(...);
    // 增强：向量检索
    String query = currentTurnUserMessage();
    List<Memory> semantic = searchByQuery(..., query, 5);
    return mergeAndDedupe(base, semantic);
}
```

## 8. 记忆冲突与去重

### 8.1 冲突策略

| 情况 | 处理 |
|------|------|
| 同 key 不同 scope | 取最具体：user_project > user_global |
| 同 key 同 scope 多次写入 | 覆盖（updated_at 最新） |
| shared 与 user 矛盾 | 注入时都列出，让 agent 选择 |

### 8.2 去重

agent 多次写同一 key 时：

```java
// 本代码仅作学习材料参考
public void upsert(Memory m) {
    Optional<Memory> existing = repo.findByUnique(
        m.tenantId(), m.userId(), m.projectId(), m.scope(), m.key()
    );
    if (existing.isPresent()) {
        Memory e = existing.get();
        e.setValue(m.value());
        e.setUpdatedAt(Instant.now());
        repo.save(e);
    } else {
        repo.save(m);
    }
}
```

## 10. 隐私与安全

- **跨租户隔离**：memory 永远带 tenant_id，查询必带；
- **不存敏感**：SaveMemory 工具调用前过 [24 章 SecretGuard](./24-智能体安全.md)；
- **可删除**：用户可一键删除自己所有记忆（GDPR right to be forgotten）；
- **可导出**：用户可导出 JSON 格式的所有记忆。

```java
// 本代码仅作学习材料参考
@GetMapping("/api/memories/export")
public ResponseEntity<byte[]> exportMyMemories() {
    List<Memory> mine = memoryService.findAllOf(currentUser());
    byte[] json = om.writeValueAsBytes(mine);
    return ResponseEntity.ok()
        .header("Content-Disposition", "attachment; filename=memories.json")
        .body(json);
}

@DeleteMapping("/api/memories/all")
public void deleteAllMyMemories() {
    memoryService.deleteAllOf(currentUser());
    // 同时 emit 审计事件
    audit.emit("user_deleted_all_memories", currentUser());
}
```

## 11. 测试

### 11.1 流程

1. 用户对话"我们用 React + TS"；
2. agent 调 SaveMemory（弹确认）；
3. 用户确认；
4. 关闭 session，开新 session；
5. 让 agent 做 todo；
6. agent 不问技术栈，直接做。

**检查点 25-1**：记忆跨 session 生效。

### 11.2 多用户隔离

1. 用户 A 存记忆；
2. 用户 B 开 session；
3. agent 看不到 A 的记忆。
