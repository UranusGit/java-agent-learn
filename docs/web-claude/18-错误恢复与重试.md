# 18 - 错误恢复与重试：让长程任务能自愈

> 长程任务跑数小时，期间必然遇到：网络抖动、模型超时、工具崩溃、沙箱 OOM、git 冲突……
> 如果每种失败都直接 abort，长程任务不可能完成。
> 本章把所有失败路径系统化，让 Agent 能从绝大多数失败中自愈。

---

| 步 | 节 | 你要带走什么 |

| ① 痛点 | §1 | 任何失败直接 abort → 长程任务根本跑不完 |

## 1. 失败分类

| 类别 | 例子 | 频率 | 默认策略 |
|------|------|------|---------|
| **网络瞬时** | TCP reset、502、连接超时 | 高 | 指数退避重试 |
| **限流** | 429、rate limit | 中 | 排队等待 |
| **模型错误** | 500、context too long、max_output_tokens | 中 | fallback / 压缩 |
| **工具失败** | exit code != 0、超时 | 中 | 让 agent 自己处理 |
| **沙箱崩溃** | 容器 OOM、被 kill | 低 | 迁移到新沙箱 |
| **持久化失败** | DB 锁、S3 抖动 | 低 | 重试 + 对账 |
| **业务错误** | 测试不通过、git 冲突 | 中 | agent 自纠 |
| **致命错误** | 配额耗尽、认证失效 | 低 | 拒绝 + 通知 |

### 1.2 错误识别

把原始异常映射到 AgentError：

```java
// 本代码仅作学习材料参考
@Component
public class ErrorClassifier {

    public AgentError classify(Throwable t) {
        if (t instanceof WebClientResponseException w) {
            return classifyHttp(w);
        }
        if (t instanceof java.util.concurrent.TimeoutException) {
            return new AgentError("network", "TIMEOUT", t.getMessage(), t, true, 1000, Map.of());
        }
        if (t instanceof ToolException te) {
            return new AgentError("tool", te.code(), te.getMessage(), t, false, 0, Map.of());
        }
        if (t instanceof OutOfMemoryError) {
            return new AgentError("fatal", "OOM", t.getMessage(), t, false, 0, Map.of());
        }
        return new AgentError("unknown", "UNKNOWN", t.getMessage(), t, false, 0, Map.of());
    }

    private AgentError classifyHttp(WebClientResponseException w) {
        return switch (w.getStatusCode().value()) {
            case 429 -> new AgentError("rate_limit", "HTTP_429",
                "rate limited", w, true, parseRetryAfter(w), Map.of());
            case 500, 502, 503, 504 -> new AgentError("network", "HTTP_5XX",
                "server error " + w.getStatusCode().value(), w, true, 1000, Map.of());
            case 401, 403 -> new AgentError("fatal", "AUTH_FAILED",
                "auth failed", w, false, 0, Map.of());
            default -> new AgentError("network", "HTTP_" + w.getStatusCode().value(),
                w.getMessage(), w, false, 0, Map.of());
        };
    }

    private long parseRetryAfter(WebClientResponseException w) {
        String ra = w.getHeaders().getFirst("Retry-After");
        if (ra == null) return 1000;
        return Long.parseLong(ra) * 1000;
    }
}
```

## 3. 重试策略

### 2.1 RetryPolicy

```java
// 本代码仅作学习材料参考
public class RetryPolicy {

    public record Decision(boolean shouldRetry, long delayMs, int nextAttempt, String reason) {
        public static Decision giveUp(String reason) {
            return new Decision(false, 0, 0, reason);
        }
        public static Decision retry(long delayMs, int nextAttempt) {
            return new Decision(true, delayMs, nextAttempt, "retry");
        }
    }

    public Decision decide(AgentError error, int attempt, int maxAttempts) {
        if (!error.retryable()) return Decision.giveUp("not retryable: " + error.code());
        if (attempt >= maxAttempts) return Decision.giveUp("max attempts reached");

        long baseDelay = error.suggestedDelayMs() > 0 ? error.suggestedDelayMs() : 1000;
        long delay = (long) (baseDelay * Math.pow(2, attempt - 1) * jitter());
        return Decision.retry(Math.min(delay, 60_000), attempt + 1);
    }

    private double jitter() {
        // 全抖动：0.5 ~ 1.5
        return 0.5 + Math.random();
    }
}
```

### 2.2 RetryTemplate

```java
// 本代码仅作学习材料参考
@Component
public class RetryTemplate {

    private final ErrorClassifier classifier;
    private final RetryPolicy policy;
    private final AgentEventBus eventBus;

    public <T> CompletableFuture<T> execute(Supplier<CompletableFuture<T>> action,
                                              String operationName,
                                              UUID sessionId,
                                              int maxAttempts) {
        CompletableFuture<T> result = new CompletableFuture<>();
        attempt(action, 1, maxAttempts, operationName, sessionId, result);
        return result;
    }

    private <T> void attempt(Supplier<CompletableFuture<T>> action,
                              int attempt, int maxAttempts,
                              String opName, UUID sessionId,
                              CompletableFuture<T> result) {
        action.get().whenComplete((value, ex) -> {
            if (ex == null) {
                result.complete(value);
                return;
            }
            AgentError err = classifier.classify(unwrap(ex));
            RetryPolicy.Decision d = policy.decide(err, attempt, maxAttempts);

            if (!d.shouldRetry()) {
                eventBus.emit(sessionId, "system", "critical",
                    Map.of("kind", "error", "category", err.category(),
                           "code", err.code(), "message", err.message(),
                           "attempts", attempt, "fatal", true)).subscribe();
                result.completeExceptionally(ex);
                return;
            }

            eventBus.emit(sessionId, "system", "info",
                Map.of("kind", "recovery", "strategy", "retry",
                       "attempt", d.nextAttempt(), "maxAttempts", maxAttempts,
                       "delayMs", d.delayMs(), "reason", err.code())).subscribe();

            scheduler.schedule(() -> attempt(action, d.nextAttempt(), maxAttempts,
                                              opName, sessionId, result),
                d.delayMs(), TimeUnit.MILLISECONDS);
        });
    }

    private Throwable unwrap(Throwable t) {
        if (t instanceof java.util.concurrent.CompletionException c) return c.getCause();
        return t;
    }
}
```

### 2.3 各模块的默认 maxAttempts

| 模块 | maxAttempts | 备注 |
|------|-------------|------|
| 模型调用 | 3 | 第 2 次开始考虑 fallback |
| 工具调用 | 0 | 工具自己负责（agent 看到 error 自己决定）|
| MCP 调用 | 3 | |
| DB 写入 | 5 | 短延迟 |
| S3 上传 | 5 | |
| WS 发送 | ∞ | 一直重试直到重连 |

## 4. 模型调用专项：fallback + max_output_tokens 恢复

### 3.1 完整流程

```java
// 本代码仅作学习材料参考
public Flux<ChatResponse> chatWithRecovery(ChatRequest req, UUID sessionId) {
    return retryTemplate.execute(
        () -> attemptModel(req, sessionId).toFuture(),
        "model_call",
        sessionId,
        3
    ).flatMapMany(this::toFlux).onErrorResume(ex -> {
        // 第 3 次失败 → fallback
        String next = fallbackChain.nextModel(req.model());
        if (next == null) return Flux.error(ex);

        eventBus.emit(sessionId, "system", "info",
            Map.of("kind", "model_fallback", "from", req.model(), "to", next,
                   "reason", ex.getMessage())).subscribe();

        ChatRequest downgraded = req.withModel(next);
        return chatWithRecovery(downgraded, sessionId);
    });
}

private Mono<ChatResponse> attemptModel(ChatRequest req, UUID sessionId) {
    return inferenceClient.chat(req)
        .onErrorResume(ex -> {
            if (isMaxOutputTokens(ex)) {
                // 恢复：stripSignatureBlocks + retry 一次
                ChatRequest cleaned = req.withMessages(stripSignatureBlocks(req.messages()));
                eventBus.emit(sessionId, "system", "info",
                    Map.of("kind", "max_output_tokens_recovery")).subscribe();
                return inferenceClient.chat(cleaned);
            }
            return Mono.error(ex);
        });
}

private boolean isMaxOutputTokens(Throwable ex) {
    return ex.getMessage() != null && ex.getMessage().contains("max_output_tokens");
}
```

## 5. 工具失败：让 Agent 自己决定

工具失败不要自动重试（agent 决定怎么处理）：

```java
// 本代码仅作学习材料参考
private void handleToolCall(ContentBlock.ToolUse tu, ...) {
    try {
        ToolResult r = tool.apply(tu.input(), ctx).join();
        // 正常处理
    } catch (Exception e) {
        // 包装成 tool_result(isError=true) 给 agent
        ToolResult errResult = ToolResult.error("tool crashed: " + e.getMessage());
        emitToolResult(tu.id(), errResult);
        // agent 看到 error，自己决定重试 / 换方案 / 放弃
    }
}
```

**理由**：
- 工具有副作用（写文件、调 API），自动重试可能造成重复写入；
- agent 看到 error 后的决策更智能（"命令找不到？我换个写法"）；
- 防止死循环（agent 一直重试同样的失败）。

## 8. 孤儿恢复

### 7.1 启动时扫孤儿

07 章提过但没实现，这里补：

```java
// 本代码仅作学习材料参考
@PostConstruct
public void recoverOrphans() {
    List<SessionEntity> orphanedSessions = sessionRepo.findAbortedWithoutEndTurn();
    for (SessionEntity s : orphanedSessions) {
        recoverSession(s);
    }
}

private void recoverSession(SessionEntity s) {
    List<Message> all = store.loadAll(s.getTenantId(), s.getId());
    List<Message> chain = store.loadLinearChain(s.getTenantId(), s.getId());

    // 检查最后一条是否是 tool_use 但没有 tool_result
    Message last = chain.get(chain.size() - 1);
    if (containsToolUse(last) && !hasFollowingResult(all, last)) {
        // 插入孤儿 tool_result
        Message recovery = Message.toolResult(last.id(),
            new ContentBlock.ToolResult(
                last.firstToolUseId(),
                "[recovered] session was interrupted, no result available",
                true
            ));
        store.append(s.getTenantId(), s.getId(), recovery);
        log.info("recovered orphan tool_result for session {}", s.getId());
    }

    s.setStatus("active");  // 让用户能继续
    sessionRepo.save(s);
}
```

## 9. 断路器

某些错误连续发生时，短路避免雪崩：

```java
// 本代码仅作学习材料参考
@Component
public class CircuitBreaker {

    private final Map<String, BreakerState> states = new ConcurrentHashMap<>();

    enum State { CLOSED, OPEN, HALF_OPEN }

    static class BreakerState {
        State state = State.CLOSED;
        int consecutiveFailures = 0;
        long openedAt = 0;
    }

    public boolean allow(String key) {
        BreakerState s = states.computeIfAbsent(key, k -> new BreakerState());
        if (s.state == State.OPEN) {
            if (System.currentTimeMillis() - s.openedAt > 60_000) {
                s.state = State.HALF_OPEN;
                return true;  // 放一个请求过去试探
            }
            return false;
        }
        return true;
    }

    public void recordSuccess(String key) {
        BreakerState s = states.get(key);
        if (s == null) return;
        s.state = State.CLOSED;
        s.consecutiveFailures = 0;
    }

    public void recordFailure(String key) {
        BreakerState s = states.computeIfAbsent(key, k -> new BreakerState());
        s.consecutiveFailures++;
        if (s.consecutiveFailures >= 5) {
            s.state = State.OPEN;
            s.openedAt = System.currentTimeMillis();
        }
    }
}
```

应用：每个 MCP server、每个租户的模型调用、每个外部 webhook 都可以套断路器。

## 12. Agent 自我纠错的引导

让 agent 看到错误后**不要无脑重试**：

system prompt 加：

```
当你调用工具失败时：
  - 仔细读错误信息；
  - 不要用同样的输入重试；
  - 改方法（换工具 / 换参数 / 换思路）；
  - 连续失败 3 次同一工具 → 停下来，向用户报告；
  - 不要把"错误"包装成"完成"。
```

## 13. 长程任务的特殊恢复

长程任务（11/13 章）每个 session 独立。如果一个 session 中途崩溃：

### 12.1 单 session 崩溃

```
session-N 跑到一半，进程被 kill
  ↓ Cron 下次 tick
发现 task 状态仍是 running，但 last_session_ended_at 超过 10 分钟
  ↓
标记 session-N 为 failed
  ↓
启动 session-N+1（Coding Agent）
  ↓
新 session 读 progress.txt + git log
发现 F012 没 commit → 重做 F012
```

### 12.2 git 状态损坏

```
worktree 处于 detached HEAD / 半 merge 状态
  ↓
重置到上一个 commit：git reset --hard HEAD
  ↓
如果还坏：从 S3 拉最近的 worktree 快照
  ↓
启动新 session
```

### 12.3 feature 卡住

```
某个 feature 测试连续 N 次不通过
  ↓
Harness Guard 标记 F0XX 为 blocked
  ↓
Planner agent 决定：
  - 跳过（标 "blocked"，做下一个）
  - 求助（在 progress.txt 写 "F0XX blocked, need human"）
  - 重做（提供更多信息后重派）
```

## 14. 验证：实战 checklist

长程任务上线前过一遍：

- [ ] 模型调用有 fallback；

- [ ] 死信队列有 worker 消费；

- [ ] 单 session 失败不导致整个 task 失败；
- [ ] worktree 损坏能从 S3 恢复。

## 15. 避坑：错误恢复常踩的雷

| 雷 | 现象 | 规避 |
|----|------|------|
| 重试风暴 | 上游恢复前疯狂重试 | 指数退避 + jitter + 上限次数 |
| 同步重试阻塞主流程 | 单请求卡死整个 Agent Loop | 异步重试 + reactive 链 |
| 断路器误开 | 偶发错误导致服务自封 | 阈值保守 + 半开探测 |
| 死信队列积压 | 没人看死信 | 监控 + 告警 + 定期人工 |
| 模型 fallback 死循环 | A→B→A→B | 严格单向 + cooldown |
| max_output_tokens 误判 | 把限流当 token 超 | 看错误码再决定（§4）|
| 孤儿恢复误删有效数据 | 把没坏的消息也修了 | 严格条件 + dry-run |
| 对账把对的说成错 | 时序竞态误报 | 容忍窗口 + 人工确认 |
| 工具失败上抛变 abort | 一次工具错毁整个任务 | §5 工具错回给 Agent，让 Agent 自纠 |
| Agent 自我纠错死循环 | 反复重试同样操作 | 同一错误 ≤2 次 + 强制换思路 |
