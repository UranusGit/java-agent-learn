# 06 - 沙箱接入：Docker 容器化执行

> 本章目标：把 05 章的 Bash 工具迁到 Docker 容器内执行，实现真正的隔离。
> 完成后：每个 session 独占一个容器，文件读写和命令执行都在容器内进行。

---

## 0. 设计回顾

- 每个 session 一个容器；
- 容器 mount 一个 worktree（绑到 `/workspace`）；
- 工具操作全部在容器内；
- session 结束 → 提取 artifact → 销毁容器。

## 1. SandboxManager

新建 `src/main/java/org/demo02/webclaude/sandbox/SandboxManager.java`：

```java
// 本代码仅作学习材料参考
package org.demo02.webclaude.sandbox;

import com.github.dockerjava.api.DockerClient;
import com.github.dockerjava.api.command.CreateContainerResponse;
import com.github.dockerjava.api.model.Bind;
import com.github.dockerjava.api.model.HostConfig;
import com.github.dockerjava.api.model.Volume;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.stereotype.Component;

import java.nio.file.Files;
import java.nio.file.Path;
import java.util.UUID;
import java.util.concurrent.ConcurrentHashMap;
import java.util.concurrent.TimeUnit;

@Component
public class SandboxManager {

    private final DockerClient docker;
    private final String image;
    private final String network;

    private final ConcurrentHashMap<UUID, Sandbox> sandboxes = new ConcurrentHashMap<>();

    public SandboxManager(DockerClient docker,
                          @Value("${webclaude.sandbox.image}") String image,
                          @Value("${webclaude.sandbox.network}") String network) {
        this.docker = docker;
        this.image = image;
        this.network = network;
    }

    public Sandbox start(UUID sessionId) {
        return sandboxes.computeIfAbsent(sessionId, id -> {
            try {
                Path worktree = Files.createTempDirectory("webclaude-wt-" + id);
                HostConfig host = HostConfig.newHostConfig()
                    .withNetworkMode(network)
                    .withBinds(new Bind(worktree.toString(), new Volume("/workspace")))
                    .withMemory(2L * 1024 * 1024 * 1024)  // 2 GB
                    .withCpuCount(2L)
                    .withSecurityOpts(java.util.List.of("no-new-privileges"))
                    .withCapDrop(java.util.List.of("ALL"));

                CreateContainerResponse resp = docker.createContainerCmd(image)
                    .withHostConfig(host)
                    .withWorkingDir("/workspace")
                    .withEnv("CLAUDE_JOB_DIR=/workspace")
                    .withCmd("sleep", "infinity")
                    .exec();
                docker.startContainerCmd(resp.getId()).exec();

                Sandbox sb = new Sandbox(id, resp.getId(), worktree);
                return sb;
            } catch (Exception e) {
                throw new RuntimeException("sandbox start failed", e);
            }
        });
    }

    public ExecResult exec(UUID sessionId, String[] command, int timeoutSec) {
        Sandbox sb = sandboxes.get(sessionId);
        if (sb == null) throw new IllegalStateException("sandbox not started: " + sessionId);

        var exec = docker.execCreateCmd(sb.containerId())
            .withAttachStdout(true)
            .withAttachStderr(true)
            .withCmd(command)
            .exec();

        StringBuilder out = new StringBuilder();
        try (var stream = docker.execStartCmd(exec.getId()).start().attach()) {
            long deadline = System.currentTimeMillis() + timeoutSec * 1000L;
            byte[] buf = new byte[4096];
            while (System.currentTimeMillis() < deadline) {
                int n = stream.read(buf);
                if (n < 0) break;
                if (n > 0) out.append(new String(buf, 0, n));
            }
        } catch (Exception e) {
            return new ExecResult(out.toString(), -1, true, "io error: " + e.getMessage());
        }

        var inspect = docker.inspectExecCmd(exec.getId()).exec();
        return new ExecResult(out.toString(), inspect.getExitCodeLong() == null ? -1 : inspect.getExitCodeLong().intValue(), false, null);
    }

    public void stop(UUID sessionId) {
        Sandbox sb = sandboxes.remove(sessionId);
        if (sb == null) return;
        try {
            docker.stopContainerCmd(sb.containerId()).withTimeout(5).exec();
            docker.removeContainerCmd(sb.containerId()).withForce(true).exec();
        } catch (Exception ignored) {}
        try {
            Files.deleteIfExists(sb.worktree());
        } catch (Exception ignored) {}
    }

    public record Sandbox(UUID sessionId, String containerId, Path worktree) {}
    public record ExecResult(String stdout, int exitCode, boolean timeout, String error) {}
}
```

## 2. DockerClient 配置

新建 `src/main/java/org/demo02/webclaude/config/DockerConfig.java`：

```java
// 本代码仅作学习材料参考
package org.demo02.webclaude.config;

import com.github.dockerjava.api.DockerClient;
import com.github.dockerjava.core.DefaultDockerClientConfig;
import com.github.dockerjava.core.DockerClientImpl;
import com.github.dockerjava.httpclient5.ApacheDockerHttpClient;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class DockerConfig {

    @Bean
    public DockerClient dockerClient() {
        var config = DefaultDockerClientConfig.createDefaultConfigBuilder().build();
        var http = new ApacheDockerHttpClient.Builder()
            .dockerHost(config.getDockerHost())
            .build();
        return DockerClientImpl.getInstance(config, http);
    }
}
```

## 3. 改造 BashTool

替换 `BashTool.apply`，让它走沙箱：

```java
// 本代码仅作学习材料参考
@Override
public CompletableFuture<ToolResult> apply(Map<String, Object> input, ToolContext ctx) {
    return CompletableFuture.supplyAsync(() -> {
        try {
            SandboxManager.ExecResult r = sandbox.exec(
                ctx.sessionId(),
                new String[]{"bash", "-c", (String) input.get("command")},
                30
            );
            if (r.timeout()) {
                return ToolResult.error("timeout after 30s");
            }
            if (r.exitCode() != 0) {
                return new ToolResult(r.stdout(), true, false, null, null);
            }
            return ToolResult.text(r.stdout());
        } catch (Exception e) {
            return ToolResult.error("sandbox exec failed: " + e.getMessage());
        }
    });
}
```

> 注入 `SandboxManager sandbox`，构造函数注入。

## 4. 改造 Read / Write / Edit / Grep

最干净的方式：把这 4 个工具也改为通过 sandbox exec 执行（用 `cat` / `tee` / `sed` / `rg`）。

但更简单的方式：因为 worktree 路径就在宿主机上（`/tmp/webclaude-wt-xxx`），宿主机也能直接访问。**生产环境**应当禁用宿主机访问（让 worktree 在容器独占的 volume 上）。本章 v1 用前者简化。

## 6. 测试沙箱

### 6.1 流程

1. 创建新 session；
2. 让 Agent 跑 `pwd && whoami && uname -a`；
3. 看到输出是 `/workspace` + `claude` + Linux 内核信息（不是你本机！）；

### 6.2 隔离验证

让 Agent 跑 `ls / && ls /workspace`：
- `/` 是容器的根（应该看到 bin / boot / dev / etc / ...）；
- `/workspace` 是 mount 进来的。

**检查点 06-1**：
- 工具命令在容器内执行；
- 容器之间的 workspace 互不可见。

## 7. 沙箱生命周期管理（关键）

### 7.1 长程任务的沙箱策略

长程任务跨多个 session，每个 session 启动一个新容器；worktree 应当**持久化**到对象存储 + 在新 session 时恢复。

```java
// 本代码仅作学习材料参考
public Sandbox startForTask(UUID sessionId, UUID taskId) {
    // 1. 从 S3 拉取 task 的 worktree 快照
    Path local = Files.createTempDirectory("webclaude-task-" + taskId);
    s3Service.downloadWorktree(taskId, local);

    // 2. 启动容器 mount 这个目录
    Sandbox sb = startWithWorktree(sessionId, local);

    // 3. 注册关闭钩子：session 结束 → 上传快照 → 销毁容器
    return sb;
```

### 7.2 容器超时清理

后台任务：每 5 分钟扫描超过 4 小时的容器，强制销毁。

```java
// 本代码仅作学习材料参考
@Scheduled(fixedRate = 5 * 60 * 1000)
public void cleanupStaleContainers() {
    sandboxes.forEach((sid, sb) -> {
        if (sb.ageMinutes() > 240) {
            System.out.println("[sandbox] force-stop stale: " + sid);
            stop(sid);
        }
    });
}
```
