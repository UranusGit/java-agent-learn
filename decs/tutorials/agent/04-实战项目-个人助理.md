## 1. 项目目标

### 1.1 业务场景

一个能用自然语言完成日常任务的助手：

```
用户："明天北京天气怎么样？要不要带伞？"
Agent：调天气 API → "明天北京有雨，建议带伞"

用户："帮我加个明天 10 点的会议，主题是产品评审"
Agent：调日历 API → "已添加"

用户："我这周还有什么待办？"
Agent：调待办 API → "3 项未完成"

用户："给张三发邮件说周五开会"
Agent：① 查张三邮箱 → ② 发邮件
```

### 1.2 技术目标

| 维度 | 要求 |
|------|------|
| Tool 数量 | 6+ 个 |
| 多轮对话 | ✅ 支持 |
| 流式输出 | ✅ 支持 |
| 防失控 | ✅ 迭代上限 + 超时 + 循环检测 |
| 评估 | ✅ 20 条测试用例 |
| 框架 | LangChain4j 或 Spring AI（自选） |

## 3. Tool 设计（基于 01 节原则）

### 3.1 WeatherTools

```java
@Component
@RequiredArgsConstructor
public class WeatherTools {

    private final WeatherService service;

    @Tool("""
        查询指定城市指定日期的天气预报。
        使用场景：用户问天气、温度、下雨、穿什么。
        返回：Weather 对象（city, date, temperature, condition, suggestion）
        """)
    public Weather getWeather(
            @ToolParam(description = "城市中文名，如 '北京'", example = "北京") String city,
            @ToolParam(description = "日期 yyyy-MM-dd，'今天'用 today", example = "2026-07-12") String date
    ) {
        return service.query(city, date);
    }
}
```

### 3.2 TodoTools

```java
@Component
@RequiredArgsConstructor
public class TodoTools {

    private final TodoService service;

    @Tool("创建一个待办事项")
    public Todo createTodo(
            @ToolParam("待办标题，简短描述") String title,
            @ToolParam("截止日期 yyyy-MM-dd") String dueDate,
            @ToolParam("优先级：HIGH / MEDIUM / LOW") String priority
    ) {
        return service.create(title, dueDate, Priority.valueOf(priority));
    }

    @Tool("列出当前用户的所有待办，可按状态过滤")
    public List<Todo> listTodos(
            @ToolParam("状态过滤：ALL / PENDING / DONE，默认 ALL") String status
    ) {
        return service.list(Status.valueOf(status));
    }

    @Tool("根据 ID 标记待办为完成")
    public Todo completeTodo(@ToolParam("待办 ID") String id) {
        return service.markDone(id);
    }
}
```

### 3.3 CalendarTools

@RequiredArgsConstructor
public class CalendarTools {

### 3.4 EmailTools

```java
@Component
@RequiredArgsConstructor
public class EmailTools {

    private final EmailService service;
    private final ContactService contactService;  // 联系人查邮箱

    @Tool("给指定收件人发邮件")
    public Email sendEmail(
            @ToolParam("收件人姓名（系统会自动查邮箱）") String recipientName,
            @ToolParam("邮件主题") String subject,
            @ToolParam("邮件正文") String body
    ) {
        String email = contactService.findEmail(recipientName);
        if (email == null) {
            throw new RuntimeException("联系人 " + recipientName + " 不存在，请确认姓名");
        }
        return service.send(email, subject, body);
    }
}
```

## 4. 业务 Service（伪造数据起步）

### 4.1 用 Map 内存存储

```java
@Service
public class TodoService {

    private final Map<String, Todo> store = new ConcurrentHashMap<>();

    public Todo create(String title, String dueDate, Priority priority) {
        String id = UUID.randomUUID().toString();
        Todo todo = new Todo(id, title, dueDate, priority, Status.PENDING);
        store.put(id, todo);
        return todo;
    }

    public List<Todo> list(Status status) {
        return store.values().stream()
                .filter(t -> status == Status.ALL || t.status() == status)
                .toList();
    }

    public Todo markDone(String id) {
        Todo t = store.get(id);
        if (t == null) throw new RuntimeException("待办 " + id + " 不存在");
        Todo updated = new Todo(t.id(), t.title(), t.dueDate(), t.priority(), Status.DONE);
        store.put(id, updated);
        return updated;
    }
}
```

### 4.2 WeatherService 接外部 API（可选）

```java
@Service
public class WeatherService {

    private final WebClient client;
    private final String apiKey;

    public Weather query(String city, String date) {
        return client.get()
                .uri(uri -> uri
                        .scheme("https")
                        .host("api.openweathermap.org")
                        .path("/data/2.5/forecast")
                        .queryParam("q", city)
                        .queryParam("appid", apiKey)
                        .queryParam("lang", "zh_cn")
                        .queryParam("units", "metric")
                        .build())
                .retrieve()
                .bodyToMono(OpenWeatherResponse.class)
                .timeout(Duration.ofSeconds(5))
                .map(r -> r.toWeather(city, date))
                .block();
    }
}
```

起步阶段用**伪造数据**，跑通后再接真实 API。

### 5.2 ChatClient 配置

```java
@Configuration
public class AiConfig {

    @Bean
    ChatClient assistantClient(
            ChatClient.Builder builder,
            WeatherTools weather,
            TodoTools todo,
            CalendarTools calendar,
            EmailTools email,
            ChatMemoryStore store
    ) {
        return builder
                .defaultSystem("""
                    你是用户的个人助理。
                    - 帮用户查询天气、管理日程、处理待办、发邮件
                    - 不确定时主动问用户细节
                    - 工具调用失败时如实告知
                    """)
                .defaultTools(weather, todo, calendar, email)
                .defaultAdvisors(
                    new MessageChatMemoryAdvisor(MessageWindowChatMemory.builder().build()),
                    new IterationLimitAdvisor(10),
                    new LoopDetectionAdvisor(),
                    new CostMonitorAdvisor(new BigDecimal("10")),
                    new SimpleLoggerAdvisor()
                )
                .build();
    }
}
```

## 6. Controller（流式）

```java
@RestController
@RequiredArgsConstructor
public class AssistantController {

    private final ChatClient assistantClient;

    @GetMapping(value = "/chat", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<String> chat(
            @RequestParam String q,
            @RequestParam(defaultValue = "user-001") String userId
    ) {
        return assistantClient.prompt()
                .user(q)
                .advisors(spec -> spec.param(ChatMemory.CONVERSATION_ID, userId))
                .stream()
                .content();
    }
}
```

## 10. 常见问题

### 10.1 LLM 不调 Tool 直接编造

**原因**：Tool 描述模糊。
**解决**：按 01 节原则重写描述。

### 10.2 LLM 调错 Tool

**原因**：Tool 描述相似度高。
**解决**：在描述里写"不适用场景"。

### 10.3 Agent 跑得慢

**原因**：多次 Tool 调用串行。
**解决**：
- 优化 Tool 内部速度
- 提高模型温度（更果断）
- 用更快模型

### 10.4 成本超预算

**解决**：
- 加 CostMonitorAdvisor
- 简单任务用便宜模型
- 缩短 ChatMemory 窗口
