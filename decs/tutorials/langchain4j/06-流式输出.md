# LangChain4j 06 - 流式输出

> 目标：理解为什么必须流式，掌握 LangChain4j 的流式 API，能在 Web 接口里实现 SSE 推流。
> 前置：已完成 01-04。

---

## 1. 为什么必须流式

### 1.1 不流式的体验灾难

```
用户点击发送 → 等 15 秒 → 屏幕突然弹出 500 字答案
```

用户体验：
- 看不到进度，以为卡死了
- 用户心理学：3 秒内没反应就会怀疑程序挂了

### 1.2 流式后的体验

```
用户点击发送 → 0.5 秒后第一个字出现 → 逐字输出 → 15 秒读完
```

用户感知**响应快**，且能边读边等。

### 1.3 技术原理

LLM 生成文本是**逐 token 输出**的（一个 token 大约 0.5-4 个汉字）。
- 不流式：等所有 token 生成完才返回完整字符串
- 流式：每生成一个 token 就立刻推送给客户端

## 2. 底层协议：SSE（Server-Sent Events）

### 2.1 SSE 是什么

> HTTP 长连接 + 服务器持续推送文本数据。客户端用 `EventSource` 接收。

不是 WebSocket。SSE 是**单向（服务器 → 客户端）**，简单、稳定、自带断线重连。

### 2.2 SSE 响应格式

```
data: 第一个token

data: 第二个

data: 个token

data: [DONE]
```

每条消息以 `data: ` 开头，以空行分隔。

### 2.3 客户端接收

```javascript
const es = new EventSource('/chat?prompt=hello');
es.onmessage = (e) => {
    document.body.innerHTML += e.data;
};
```

## 3. LangChain4j 的流式 API

### 3.1 两种模型类型

```java
// 非流式（你前面用的）
import dev.langchain4j.model.openai.OpenAiChatModel;

// 流式
import dev.langchain4j.model.openai.OpenAiStreamingChatModel;
```

**区别**：流式模型的方法返回 `void`，通过回调接收数据。

### 3.2 最简单的流式调用

```java
package org.demo01.streaming;

import dev.langchain4j.model.StreamingResponseHandler;
import dev.langchain4j.model.openai.OpenAiStreamingChatModel;

public class StreamDemo {
    public static void main(String[] args) throws InterruptedException {
        var model = OpenAiStreamingChatModel.builder()
                .baseUrl("https://api.deepseek.com")
                .apiKey(System.getenv("DEEPSEEK_API_KEY"))
                .modelName("deepseek-chat")
                .temperature(0.7)
                .build();

        model.chat("讲一个长一点的笑话", new StreamingResponseHandler<>() {
            @Override
            public void onPartialResponse(String partialResponse) {
                System.out.print(partialResponse);
            }

            @Override
            public void onCompleteResponse(String completeResponse) {
                System.out.println("\n[完成]");
            }

            @Override
            public void onError(Throwable error) {
                error.printStackTrace();
            }
        });

        // 流式是异步的，主线程要等一下
        Thread.sleep(30_000);
    }
}
```

### 3.3 TokenStream：AiServices 的流式接口

更常用的方式：在 AiServices 接口里返回 `TokenStream`。

```java
import dev.langchain4j.service.TokenStream;

public interface StreamingAssistant {
    TokenStream chat(String userMessage);
}

StreamingAssistant agent = AiServices.builder(StreamingAssistant.class)
        .streamingChatLanguageModel(streamingModel)
        .chatMemory(MessageWindowChatMemory.withMaxMessages(10))
        .build();

TokenStream stream = agent.chat("讲个故事");
stream.onPartialResponse(System.out::print)
      .onCompleteResponse(r -> System.out.println("\n[done]"))
      .onError(Throwable::printStackTrace)
      .start();
```

**优势**：AiServices 自动处理 memory、Tool 调用，你只关心流式回调。

## 4. Spring Boot + SSE 实战

### 4.1 依赖

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
</dependency>
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-webflux</artifactId>
</dependency>
```

### 4.2 Controller

```java
@RestController
public class ChatController {

    private final StreamingAssistant agent;

    public ChatController(StreamingAssistant agent) {
        this.agent = agent;
    }

    @GetMapping(value = "/chat", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public SseEmitter chat(@RequestParam String prompt) {
        // 超时 5 分钟
        SseEmitter emitter = new SseEmitter(300_000L);

        TokenStream stream = agent.chat(prompt);
        stream.onPartialResponse(token -> {
                try {
                    emitter.send(token);
                } catch (IOException e) {
                    emitter.completeWithError(e);
                }
            })
            .onCompleteResponse(response -> emitter.complete())
            .onError(emitter::completeWithError)
            .start();

        return emitter;
    }
}
```

### 4.3 前端测试（HTML）

```html
<!DOCTYPE html>
<html>
<body>
    <input id="prompt" placeholder="问点什么...">
    <button onclick="ask()">发送</button>
    <pre id="output"></pre>

    <script>
    function ask() {
        const prompt = document.getElementById('prompt').value;
        const output = document.getElementById('output');
        output.textContent = '';

        const es = new EventSource(`/chat?prompt=${encodeURIComponent(prompt)}`);
        es.onmessage = (e) => {
            output.textContent += e.data;
        };
        es.onerror = () => es.close();
    }
    </script>
</body>
</html>
```

## 5. WebFlux 响应式实现（推荐）

### 5.1 为什么用 WebFlux

`SseEmitter` 是 Servlet 模式的同步阻塞。WebFlux 的 `Flux` 是响应式的，**资源占用更低**，更适合长连接场景。

### 5.2 Controller

```java
@RestController
public class ChatController {

    private final StreamingAssistant agent;

    public ChatController(StreamingAssistant agent) {
        this.agent = agent;
    }

    @GetMapping(value = "/chat", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<String> chat(@RequestParam String prompt) {
        return Flux.create(sink -> {
            TokenStream stream = agent.chat(prompt);
            stream.onPartialResponse(sink::next)
                  .onCompleteResponse(response -> sink.complete())
                  .onError(sink::error)
                  .start();
        });
    }
}
```

### 5.3 关键点

- `Flux<String>` 对应 SSE 流
- `sink.next()` 推一个 token
- `sink.complete()` 结束流
- Spring 自动把 Flux 序列化成 SSE 格式

## 6. 流式 + Tool 调用

### 6.1 注意事项

LangChain4j 流式 + Tool 组合**有限制**：
- Tool 调用过程不会流式输出（因为 LLM 在生成 tool_calls，不是文字）
- 只有最终自然语言回答才会流式

### 6.2 体验

```java
StreamingAssistant agent = AiServices.builder(StreamingAssistant.class)
        .streamingChatLanguageModel(streamingModel)
        .tools(new TimeTools())
        .build();

// 用户问"现在几点"
// 内部：LLM 决定调 Tool → 执行 → LLM 生成"现在是 X" → 流式输出
agent.chat("现在几点？");
```

## 7. 常见错误

### 7.1 流式中途断开

**症状**：客户端突然收不到 token 了。
**原因**：网络抖动 / 服务器超时 / LLM 服务断开。
**解决**：
1. 客户端 EventSource 自带重连，但要注意**幂等性**
2. 服务端加超时监控：超过 60 秒没收到 token 就主动断开
3. 错误捕获后 `emitter.completeWithError()` 不要静默吞

### 7.2 中文乱码

**症状**：流式输出时半个汉字（UTF-8 多字节被分割）。
**原因**：SSE 默认按字节推，汉字可能被切断。
**解决**：
- LangChain4j 内部已经处理（按字符推而非字节）
- 如果是自定义 SSE 实现，注意字符集编码

### 7.3 `Flux` 不工作

**症状**：浏览器一直 pending。
**原因**：Spring MVC（非 WebFlux）模式下，Flux 会被当作普通集合处理。
**解决**：
- 加 `spring-boot-starter-webflux`
- 或显式指定 `produces = MediaType.TEXT_EVENT_STREAM_VALUE`

### 7.4 主线程结束太快

**症状**：本地测试时没看到任何输出。
**原因**：流式是异步的，主线程跑完了异步任务还没执行。
**解决**：`Thread.sleep` 或用 `CountDownLatch` 等待。

## 8. 性能与成本优化

### 8.1 首字延迟（TTFT）

> Time To First Token —— 用户看到第一个字的时间。

| 优化手段 | 效果 |
|---------|------|
| 用更快的小模型 | 显著 |
| 简化 prompt | 显著 |
| 启用 vLLM 服务端 KV Cache | 显著（生产） |
| 减小 max_tokens | 无影响（与首字无关） |

### 8.2 客户端节流

```javascript
// 简单节流：每 50ms 更新一次 UI
let buffer = '';
let lastUpdate = 0;

es.onmessage = (e) => {
    buffer += e.data;
    const now = Date.now();
    if (now - lastUpdate > 50) {
        output.textContent = buffer;
        lastUpdate = now;
    }
};
```

减少 DOM 操作频率，提升流畅度。
