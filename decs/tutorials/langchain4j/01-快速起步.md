# LangChain4j 01 - 快速起步

> 目标：30 分钟内跑通第一个 Java 调用 LLM 的程序。
> 前置条件：JDK 21 + Maven 已装 + 一个可用的模型服务。

---

## 0. 本教程的环境约定（重要，所有章节通用）

为统一所有示例代码，本教程系列约定如下环境：

| 角色 | 服务 | baseUrl | apiKey | 模型名 |
|------|------|---------|--------|--------|
| **对话模型（Chat）** | 远端 DeepSeek API | `https://api.deepseek.com` | 环境变量 `DEEPSEEK_API_KEY` | `deepseek-chat` |
| **Embedding 模型** | 本地 LM Studio | `http://127.0.0.1:1234/v1` | `lm-studio`（随便填） | `text-embedding-bge-large-zh-v1.5` |
| **本地对话模型（可选）** | 本地 LM Studio | `http://127.0.0.1:1234/v1` | `lm-studio` | LM Studio 里加载的模型 id |

**为什么这么组合**：
- DeepSeek 远端做 Chat —— 便宜、稳定、不需要本地 GPU
- LM Studio 做 Embedding —— 离线、零成本、隐私好；RAG 检索阶段调用量大但模型小，本地跑最划算
- DeepSeek 和 LM Studio 都暴露 OpenAI 兼容协议，所以**只需要一个依赖**：`langchain4j-open-ai`

**LM Studio 准备**：
1. 下载 embedding 模型 `text-embedding-bge-large-zh-v1.5`
2. 进入 Developer → Start Server（默认端口 `1234`）
3. 验证：`curl http://127.0.0.1:1234/v1/models` 能列出模型

**DeepSeek 准备**：
1. 注册 `platform.deepseek.com`，充值 10 元（够学一周）
2. 创建 API Key
3. 设置环境变量：
   ```bash
   export DEEPSEEK_API_KEY=sk-xxxxx   # Mac/Linux
   set  DEEPSEEK_API_KEY=sk-xxxxx     # Windows cmd
   ```
   或在 IDEA Run Configuration → Environment variables 里加 `DEEPSEEK_API_KEY=sk-xxxxx`

> **不需要 Ollama**。本教程全程不依赖 Ollama，所有"本地模型"需求由 LM Studio 满足。

## 1. LangChain4j 是什么

### 1.1 一句话定义

> LangChain4j 是 LangChain（Python）的 Java 移植版，致力让 Java 工程师用熟悉的方式开发 LLM 应用。

### 1.2 它帮你解决了什么

不用 LangChain4j 时，调一个 OpenAI 兼容接口你至少要：
- 自己拼 HTTP 请求体（messages 数组）
- 自己解析 SSE 流
- 自己管理对话历史
- 自己实现 Function Calling 协议
- 换个模型供应商要重写一半代码

LangChain4j 把这些封装成 **统一的 Java API**，换模型只改一个 Builder。

### 1.3 核心抽象（5 个必记住）

| 抽象 | 作用 | Java 类比 |
|------|------|----------|
| `ChatModel` | 调用 LLM 的入口 | `HttpClient` |
| `ChatMemory` | 对话历史管理 | `HttpSession` |
| `@Tool` | 工具方法声明 | Spring 的 `@RequestMapping` |
| `AiServices` | 声明式 Agent 接口 | MyBatis 的 Mapper 接口 |
| `ContentRetriever` | RAG 检索器 | `ResultSetExtractor` |

> **1.0 API 变化**：早期版本的 `ChatLanguageModel` 在 1.0 起统一改为 `ChatModel`（旧名仍兼容但已弃用）。本教程统一用新名。

## 2. 项目搭建

### 2.1 完整 pom.xml

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
                            http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <groupId>org.example</groupId>
    <artifactId>demo01</artifactId>
    <version>1.0-SNAPSHOT</version>

    <properties>
        <maven.compiler.source>21</maven.compiler.source>
        <maven.compiler.target>21</maven.compiler.target>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
        <langchain4j.version>1.0.1</langchain4j.version>
    </properties>

    <dependencies>
        <!-- 核心库：含 AiServices、ChatMemory、InMemoryEmbeddingStore 等 -->
        <dependency>
            <groupId>dev.langchain4j</groupId>
            <artifactId>langchain4j</artifactId>
            <version>${langchain4j.version}</version>
        </dependency>

        <!-- OpenAI 兼容协议：一个包同时支持 chat + embedding，DeepSeek 和 LM Studio 都用它 -->
        <dependency>
            <groupId>dev.langchain4j</groupId>
            <artifactId>langchain4j-open-ai</artifactId>
            <version>${langchain4j.version}</version>
        </dependency>
    </dependencies>
</project>
```

### 2.2 关键决策点

**Q：为什么只需要两个依赖？**
A：DeepSeek 远端和 LM Studio 本地都走 OpenAI 兼容协议，`langchain4j-open-ai` 一个包提供 `OpenAiChatModel` + `OpenAiEmbeddingModel`，覆盖所有需求。不需要 `langchain4j-ollama`。

**Q：版本号 `1.0.1` 哪里查？**
A：Maven Central 搜 `langchain4j`，或访问 `github.com/langchain4j/langchain4j` 的 releases。**注意：1.0 之后 API 有过重构，老教程（0.x）的代码可能跑不通**。

**Q：日志怎么看？**
A：LangChain4j 内部走 SLF4J，需要提供具体实现（logback / slf4j-simple）。本教程默认配合 Spring Boot starter（自带 logback）使用，详见 09-常见错误。

## 3. 第一个 Main：调用 LLM

```java
package org.demo01;

import dev.langchain4j.model.openai.OpenAiChatModel;

public class Main {
    public static void main(String[] args) {
        var model = OpenAiChatModel.builder()
                .baseUrl("https://api.deepseek.com")           // 远端 DeepSeek
                .apiKey(System.getenv("DEEPSEEK_API_KEY"))     // 从环境变量读
                .modelName("deepseek-chat")
                .temperature(0.7)
                .build();

        String answer = model.chat("用一句话解释什么是 RAG");
        System.out.println(answer);
    }
}
```

### 3.1 运行

```bash
# 1. 确认环境变量
echo $DEEPSEEK_API_KEY

# 2. 运行
mvn compile exec:java -Dexec.mainClass="org.demo01.Main"
```

或者在 IDEA 直接 run `Main.main()`，记得在 Run Configuration 里设 `DEEPSEEK_API_KEY` 环境变量。

## 4. 这 5 行代码到底发生了什么（必懂）

### 4.1 `ChatModel.chat(String)` 内部做了什么

```
你的代码
   ↓ "用一句话解释什么是 RAG"
LangChain4j 封装成 HTTP 请求体：
   POST https://api.deepseek.com/v1/chat/completions
   Authorization: Bearer ${DEEPSEEK_API_KEY}
   {
     "model": "deepseek-chat",
     "messages": [
       {"role":"user","content":"用一句话解释什么是 RAG"}
     ],
     "temperature": 0.7,
     "stream": false
   }
   ↓
DeepSeek 服务接收，调度模型推理
   ↓
返回响应 JSON
   ↓
LangChain4j 解析响应，提取 message.content
   ↓
返回 String 给你
```

### 4.2 接口层级

```
ChatModel（接口）
   ├── String chat(String userMessage)              // 最简单
   ├── ChatResponse chat(ChatRequest)               // 拿完整响应（含 token 用量）
   └── void chat(String, StreamingResponseHandler)  // 流式（下个教程讲）

        实现
   ├── OpenAiChatModel       ← DeepSeek 和 LM Studio 都走这个
   ├── OllamaChatModel       ← 不用，本教程不依赖 Ollama
   ├── AnthropicChatModel
   ├── QianFanChatModel
   └── ...30+ 供应商
```

### 4.3 关键设计哲学

> **同一接口，多实现**。这是 LangChain4j 的核心价值。
> 你换模型，业务代码（`model.chat(...)`）一行都不用改，只改 Builder。

类比 Spring：`ChatModel` 就像 `DataSource`，OpenAI / DeepSeek 实现就像 HikariCP/Druid。

## 5. 常用参数详解

```java
var model = OpenAiChatModel.builder()
        .baseUrl("https://api.deepseek.com")
        .apiKey(System.getenv("DEEPSEEK_API_KEY"))
        .modelName("deepseek-chat")

        // 生成参数（影响输出质量）
        .temperature(0.7)        // 0=最严谨, 1=最发散, 默认 0.7
        .topP(0.9)               // nucleus sampling，0-1
        // .topK(40)             // OpenAI 协议不传 topK（部分实现忽略）

        // 长度控制
        .maxOutputTokens(1024)   // 单次最大输出 token

        // 超时
        .timeout(Duration.ofMinutes(2))

        // 可观测（强烈推荐开启）
        .logRequests(true)       // 打印请求体（DEBUG 级别）
        .logResponses(true)      // 打印响应体（DEBUG 级别）
        .build();
```

### 5.1 参数选择经验

| 场景 | temperature | 说明 |
|------|------------|------|
| 代码生成 | 0.2 | 越严谨越好 |
| 客服对话 | 0.5-0.7 | 平稳 |
| 创意写作 | 0.9+ | 越发散越好 |
| Tool 调用 | 0.0-0.3 | 必须严谨，避免乱调 |

### 5.2 调试技巧

**初期一定要开 `logRequests(true)` 和 `logResponses(true)`**，让你看到 LangChain4j 实际发了什么、收到了什么。这是排查 90% 问题的关键。

> 注意：这两个开关走 SLF4J 的 DEBUG 级别，**必须提供 SLF4J 实现**（logback / slf4j-simple）并把 `dev.langchain4j` 包级别开到 DEBUG 才能看到。详见 09-常见错误。

## 6. 切换到本地 LM Studio（可选）

如果想完全离线（不用 DeepSeek），改成调用 LM Studio：

```java
var model = OpenAiChatModel.builder()
        .baseUrl("http://127.0.0.1:1234/v1")  // LM Studio 默认地址
        .apiKey("lm-studio")                  // LM Studio 不校验，随便填
        .modelName("qwen2.5-7b-instruct")     // LM Studio 里实际加载的模型 id
        .temperature(0.7)
        .build();
```

> 模型名要和 LM Studio Developer 面板里 `GET /v1/models` 返回的 `id` 完全一致（区分大小写、版本号、后缀）。

## 7. 常见错误排查

### 7.1 DeepSeek 连接失败 / 401

```
Connection refused: api.deepseek.com
或
{"error":{"message":"Invalid API key"}}
```

**排查**：
1. `curl https://api.deepseek.com/v1/models -H "Authorization: Bearer $DEEPSEEK_API_KEY"` 看能不能通
2. `echo $DEEPSEEK_API_KEY` 确认环境变量已设置
3. IDEA 启动时要重新打开（环境变量改动需要重启 IDEA）

### 7.2 LM Studio 连不上 / 模型名不对

```
Connection refused: localhost:1234
或
model 'qwen2.5-7b-instruct' not found
```

**排查**：
1. 浏览器访问 `http://127.0.0.1:1234/v1/models`，确认服务在线
2. 返回的 JSON 里 `id` 字段就是 `modelName` 该填的值（区分大小写、后缀）
3. LM Studio → Developer 面板里必须**已加载**模型（不只是下载）

### 7.3 baseUrl 末尾斜杠

**坑点**：`baseUrl("https://api.deepseek.com/")`（末尾带 `/`）和 `baseUrl("https://api.deepseek.com")`（不带）**有时会引发 404**。
**解决**：**不要带末尾 `/`**，LangChain4j 会自动拼 `/v1/chat/completions`。

### 7.4 中文乱码

**Windows 命令行常见**。
**解决**：`chcp 65001` 切换 UTF-8。或者在 IDEA Run Configuration 里设 VM options `-Dfile.encoding=UTF-8`。

### 7.5 logRequests 开了但看不到日志

**原因**：缺 SLF4J 实现。
**解决**：引入 `logback-classic` 或配合 Spring Boot starter（自带 logback）。详见 09-常见错误。

### 7.6 模型回复乱码或不完整

**原因**：`maxOutputTokens` 太小。
**解决**：调大到 1024+。
