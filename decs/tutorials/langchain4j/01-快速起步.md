# LangChain4j 01 - 快速起步

> 目标：30 分钟内跑通第一个 Java 调用 LLM 的程序。
> 前置条件：JDK 21 + Maven 已装 + 一个可用的模型服务（Ollama 或 DeepSeek）。

---

## 1. LangChain4j 是什么

### 1.1 一句话定义

> LangChain4j 是 LangChain（Python）的 Java 移植版，致力让 Java 工程师用熟悉的方式开发 LLM 应用。

### 1.2 它帮你解决了什么

不用 LangChain4j 时，调一个 OpenAI 兼容接口你至少要：
- 自己拼 HTTP 请求体（messages 数组）
- 自己解析 SSE 流
- 自己管理对话历史
- 自己实现 Function Calling 协议
- 换个模型供应商要重写一半代码

LangChain4j 把这些封装成 **统一的 Java API**，换模型只改一个 Builder。

### 1.3 核心抽象（5 个必记住）

| 抽象 | 作用 | Java 类比 |
|------|------|----------|
| `ChatLanguageModel` | 调用 LLM 的入口 | `HttpClient` |
| `ChatMemory` | 对话历史管理 | `HttpSession` |
| `@Tool` | 工具方法声明 | Spring 的 `@RequestMapping` |
| `AiServices` | 声明式 Agent 接口 | MyBatis 的 Mapper 接口 |
| `ContentRetriever` | RAG 检索器 | `ResultSetExtractor` |

## 2. 模型服务准备（二选一）

### 方案 A：Ollama（推荐，零成本）

```bash
# Mac 安装
brew install ollama

# 启动服务
ollama serve

# 另开一个终端拉模型（首次约 4.7GB）
ollama pull qwen2.5:7b

# 测试一下
ollama run qwen2.5:7b "你好"
```

**验证服务可用**：
```bash
curl http://localhost:11434/api/chat -d '{
  "model": "qwen2.5:7b",
  "messages": [{"role":"user","content":"hi"}],
  "stream": false
}'
```

### 方案 B：DeepSeek API（推荐，便宜稳定）

1. 注册 `platform.deepseek.com`
2. 充值 10 元（够你学一周）
3. 创建 API Key 保存好

**验证可用**：
```bash
curl https://api.deepseek.com/v1/chat/completions \
  -H "Authorization: Bearer $YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "deepseek-chat",
    "messages": [{"role":"user","content":"hi"}]
  }'
```

## 3. 项目搭建（手搓 pom.xml）

```xml

        <dependency>
            <groupId>dev.langchain4j</groupId>

    </dependencies>

    <build>
        <plugins>
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-compiler-plugin</artifactId>
                <version>3.13.0</version>
                <configuration>
                    <release>21</release>
                </configuration>
            </plugin>
        </plugins>
    </build>
</project>
```

### 3.2 关键决策点（手搓时要懂的）

**Q：为什么不用 `langchain4j-bom`？**
A：BOM 是用来管理多模块版本一致性的，对你目前单模块项目意义不大。等你引入 RAG、Tool 等多个模块时再换 BOM。

**Q：版本号 `1.0.1` 哪里查？**
A：Maven Central 搜 `langchain4j`，或访问 `github.com/langchain4j/langchain4j` 的 releases。**注意：1.0 之后 API 有过重构，老教程（0.x）的代码可能跑不通**。

**Q：为啥只引入这两个？**
A：LangChain4j 是模块化设计，核心库 + 各供应商的集成包分开。你不需要一开始就引入 RAG、向量库那些，按需引入。

## 4. 第一个 Main：调用 LLM

### 4.1 方案 A：用 Ollama

```java
package org.example;

import dev.langchain4j.model.ollama.OllamaChatModel;

public class Main {
    public static void main(String[] args) {
        var model = OllamaChatModel.builder()
                .baseUrl("http://localhost:11434")
                .modelName("qwen2.5:7b")
                .temperature(0.7)
                .build();

        String answer = model.chat("用一句话解释什么是 RAG");
        System.out.println(answer);
    }
}
```

### 4.2 方案 B：用 DeepSeek

```java
package org.example;

import dev.langchain4j.model.openai.OpenAiChatModel;

public class Main {
    public static void main(String[] args) {
        var model = OpenAiChatModel.builder()
                .baseUrl("https://api.deepseek.com")
                .apiKey(System.getenv("DEEPSEEK_API_KEY"))  // 不要硬编码
                .modelName("deepseek-chat")
                .temperature(0.7)
                .build();

        String answer = model.chat("用一句话解释什么是 RAG");
        System.out.println(answer);
    }
}
```

### 4.3 运行（命令行设环境变量）

```bash
# DeepSeek 方式需要先 export API Key
export DEEPSEEK_API_KEY=sk-xxxxx

# 运行
mvn compile exec:java -Dexec.mainClass="org.example.Main"
```

或者在 IDEA 的 Run Configuration 里加环境变量。

## 5. 这个 5 行代码到底发生了什么（必懂）

### 5.1 `ChatLanguageModel.chat(String)` 内部做了什么

```
你的代码
   ↓ "用一句话解释什么是 RAG"
LangChain4j 封装成 HTTP 请求体：
   POST http://localhost:11434/api/chat
   {
     "model": "qwen2.5:7b",
     "messages": [
       {"role":"user","content":"用一句话解释什么是 RAG"}
     ],
     "stream": false
   }
   ↓
Ollama 服务接收，调用本地模型推理
   ↓
返回响应 JSON
   ↓
LangChain4j 解析响应，提取 message.content
   ↓
返回 String 给你
```

### 5.2 接口层级

```
ChatLanguageModel（接口）
   ├── String chat(String userMessage)              // 最简单
   ├── ChatResponse chat(ChatRequest)               // 拿完整响应（含 token 用量）
   └── void chat(String, StreamingResponseHandler)  // 流式（下个教程讲）

        实现
   ├── OllamaChatModel      ← 你现在用的
   ├── OpenAiChatModel      ← DeepSeek 也走这个
   ├── AnthropicChatModel
   ├── QianFanChatModel
   └── ...30+ 供应商
```

### 5.3 关键设计哲学

> **同一接口，多实现**。这是 LangChain4j 的核心价值。
> 你换模型，业务代码（`model.chat(...)`）一行都不用改，只改 Builder。

类比 Spring：`ChatLanguageModel` 就像 `DataSource`，Ollama/OpenAI 实现就像 HikariCP/Druid。

## 6. 常用参数详解

```java
var model = OllamaChatModel.builder()
        .baseUrl("http://localhost:11434")
        .modelName("qwen2.5:7b")

        // 生成参数（影响输出质量）
        .temperature(0.7)        // 0=最严谨, 1=最发散, 默认 0.7
        .topP(0.9)               // nucleus sampling，0-1
        .topK(40)                // 只在 top-K token 中选，整数

        // 长度控制
        .maxOutputTokens(1024)   // 单次最大输出 token
        // .maxInputTokens(...)  // 输入上限（部分模型支持）

        // 超时
        .timeout(Duration.ofMinutes(2))

        // 可观测（强烈推荐开启）
        .logRequests(true)       // 打印请求体
        .logResponses(true)      // 打印响应体
        .build();
```

### 6.1 参数选择经验

| 场景 | temperature | 说明 |
|------|------------|------|
| 代码生成 | 0.2 | 越严谨越好 |
| 客服对话 | 0.5-0.7 | 平稳 |
| 创意写作 | 0.9+ | 越发散越好 |
| Tool 调用 | 0.0-0.3 | 必须严谨，避免乱调 |

### 6.2 调试技巧

**初期一定要开 `logRequests(true)` 和 `logResponses(true)`**，让你看到 LangChain4j 实际发了什么、收到了什么。这是排查 90% 问题的关键。

## 7. 常见错误排查

### 7.1 连接失败

```
Connection refused: /127.0.0.1:11434
```

**原因**：Ollama 服务没起来，或端口不对。
**解决**：
```bash
ollama serve           # 看是否启动
lsof -i :11434         # 看端口占用
```

### 7.2 401 Unauthorized

```
{"error":{"message":"Invalid API key"}}
```

**原因**：API Key 错或没设环境变量。
**解决**：`echo $DEEPSEEK_API_KEY` 看一下。IDEA 启动时要重新打开（环境变量改动需要重启 IDEA）。

### 7.3 模型未加载

```
model 'qwen2.5:7b' not found
```

**原因**：本地没拉这个模型。
**解决**：`ollama pull qwen2.5:7b`。

### 7.4 中文乱码

**Windows 命令行常见**。
**解决**：`chcp 65001` 切换 UTF-8。或者在 IDEA Run Configuration 里设 VM options `-Dfile.encoding=UTF-8`。

### 7.5 依赖冲突 / ClassNotFoundException

**原因**：版本不一致或没用 `1.x`。
**解决**：清缓存重新拉：
```bash
mvn dependency:purge-local-repository -DmanualInclude="dev.langchain4j:*"
mvn clean compile
```

### 7.6 OpenAI 模块调用 DeepSeek 时 baseUrl 末尾斜杠

**坑点**：`baseUrl("https://api.deepseek.com/")`（末尾带 `/`）和 `baseUrl("https://api.deepseek.com")`（不带）**有时会引发 404**。
**解决**：**不要带末尾 `/`**，LangChain4j 会自动拼 `/v1/chat/completions`。

### 7.7 模型回复乱码或不完整

**原因**：`maxOutputTokens` 太小。
**解决**：调大到 1024+。
