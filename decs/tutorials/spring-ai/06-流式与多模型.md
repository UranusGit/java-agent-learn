## 1. 流式输出：Spring AI 方式

### 1.1 与 LangChain4j 对比

| 维度 | LangChain4j | Spring AI |
|------|-------------|-----------|
| API | `TokenStream` + 回调 | **`Flux<String>` 响应式** |
| 集成 | 手动 SSE | Spring WebFlux 原生 |
| Stream Advisor | 弱 | 强（`aroundStream`） |
| Spring 整合 | 需自己写 | 开箱即用 |

**结论**：Spring AI 的流式**比 LangChain4j 更优雅**，因为底层就是 WebFlux 响应式。

## 2. 第一个流式接口

### 2.1 Controller

```java
@RestController
public class StreamController {

    private final ChatClient chatClient;

    public StreamController(ChatClient chatClient) {
        this.chatClient = chatClient;
    }

    @GetMapping(value = "/chat/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<String> stream(@RequestParam String q) {
        return chatClient.prompt()
                .user(q)
                .stream()
                .content();
    }
}
```

### 2.2 关键点

- `.stream()` 替代 `.call()` —— 切换到流式
- 返回类型是 `Flux<String>`
- `produces = TEXT_EVENT_STREAM_VALUE` —— 自动序列化成 SSE

### 2.3 LangChain4j 对比

```java
// LangChain4j：手动拼 SSE
@GetMapping(value = "/chat", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
public SseEmitter chat(@RequestParam String q) {
    SseEmitter emitter = new SseEmitter(300_000L);
    TokenStream stream = agent.chat(q);
    stream.onPartialResponse(token -> emitter.send(token))
          .onCompleteResponse(r -> emitter.complete())
          .onError(emitter::completeWithError)
          .start();
    return emitter;
}

// Spring AI：直接返回 Flux
@GetMapping(value = "/chat/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
public Flux<String> stream(@RequestParam String q) {
    return chatClient.prompt().user(q).stream().content();
}
```

**Spring AI 代码量 1/3**。

## 3. 流式 + Advisor

### 3.1 Advisor 自动支持流式

回顾第 02 节：Advisor 实现 `StreamAdvisor` 接口的 `aroundStream` 方法即可。

**`MessageChatMemoryAdvisor`**、**`QuestionAnswerAdvisor`** 都已经实现了流式版本，直接用就行。

### 3.2 流式 + RAG + Memory 一体

```java
@GetMapping(value = "/chat/stream", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
public Flux<String> stream(@RequestParam String q, @RequestParam String userId) {
    return chatClient.prompt()
            .user(q)
            .advisors(spec -> spec.param(ChatMemory.CONVERSATION_ID, userId))
            .stream()
            .content();
}
```

RAG、Memory、流式全部自动协作。

## 5. 多模型并存架构

### 5.1 需求

生产场景常见：
- 对话用便宜模型（如 `deepseek-chat`）
- 复杂推理用强模型（如 `deepseek-reasoner`）
- 代码生成用专门模型（如 `qwen2.5-coder`）
- 总结用 `gpt-4o-mini`

Spring AI 怎么搞？

### 5.2 方案 1：多 ChatModel Bean

```java
@Configuration
public class MultiModelConfig {

    @Bean("dialogModel")
    public OpenAiChatModel dialogModel() {
        return OpenAiChatModel.builder()
                .openAiApi(OpenAiApi.builder()
                    .baseUrl("https://api.deepseek.com")
                    .apiKey(System.getenv("DEEPSEEK_API_KEY"))
                    .build())
                .defaultOptions(OpenAiChatOptions.builder()
                    .model("deepseek-chat")
                    .build())
                .build();
    }

    @Bean("reasonerModel")
    public OpenAiChatModel reasonerModel() {
        return OpenAiChatModel.builder()
                .openAiApi(...)
                .defaultOptions(OpenAiChatOptions.builder()
                    .model("deepseek-reasoner")
                    .build())
                .build();
    }
}
```

### 5.3 方案 2：用 ChatClient.Builder 派生

```java
@Bean("dialogClient")
public ChatClient dialogClient(@Qualifier("dialogModel") ChatModel model) {
    return ChatClient.builder(model).build();
}

@Bean("reasonerClient")
public ChatClient reasonerClient(@Qualifier("reasonerModel") ChatModel model) {
    return ChatClient.builder(model).build();
}
```

### 5.4 业务层注入

```java
@Service
public class SmartAssistant {

    private final ChatClient dialogClient;
    private final ChatClient reasonerClient;

    public SmartAssistant(
        @Qualifier("dialogClient") ChatClient dialogClient,
        @Qualifier("reasonerClient") ChatClient reasonerClient
    ) {
        this.dialogClient = dialogClient;
        this.reasonerClient = reasonerClient;
    }

    public String chat(String question) {
        // 先用便宜模型判断问题类型
        String category = dialogClient.prompt()
                .system("分类问题：simple / complex")
                .user(question)
                .call()
                .content();

        if ("complex".equals(category.trim())) {
            return reasonerClient.prompt().user(question).call().content();
        }
        return dialogClient.prompt().user(question).call().content();
    }
}
```

**价值**：复杂问题走强模型，简单问题走便宜模型，**显著降本**。

### 7.2 故障切换逻辑

```java
public String robustChat(String q) {
    try {
        // 先走云端（质量好）
        return cloudClient.prompt().user(q).call().content();
    } catch (Exception e) {
        log.warn("Cloud model failed, falling back to local", e);
        // 降级到本地（永远可用）
        return localClient.prompt().user(q).call().content();
    }
}
```

**应用**：
- 网络抖动容错
- API 余额用尽降级
- 数据敏感场景切换到本地

## 9. 性能与监控

### 9.1 流式延迟监控

```java
@GetMapping("/chat/stream")
public Flux<String> stream(@RequestParam String q) {
    long start = System.currentTimeMillis();
    return chatClient.prompt()
            .user(q)
            .stream()
            .content()
            .doOnNext(token -> {
                if (System.currentTimeMillis() - start < 1000) {
                    log.info("TTFT < 1s");
                }
            })
            .doOnComplete(() -> log.info("Total: {}ms", System.currentTimeMillis() - start));
}
```

## 10. 常见错误

### 10.1 流式返回但浏览器一直转

**原因**：`produces` 没设。
**解决**：`produces = MediaType.TEXT_EVENT_STREAM_VALUE`。

### 10.2 流式 + Advisor 不生效

**原因**：Advisor 只实现了 `CallAdvisor`，没实现 `StreamAdvisor`。
**解决**：两个接口都实现。

### 10.3 多模型 Bean 冲突

**症状**：`NoUniqueBeanDefinitionException`
**解决**：用 `@Qualifier` 指定。

### 10.4 本地模型 + 云端模型 embedding 维度不一致

**症状**：写入向量库报错。
**解决**：一个 collection 一个 embedding 模型，不要混用。
