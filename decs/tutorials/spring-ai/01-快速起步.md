# Spring AI 01 - 快速起步

> 前提：你已经吃透 LangChain4j 的 5 个核心抽象。
> 本节重点：Spring Boot 集成、ChatClient 第一次调用、**与 LangChain4j 的关键差异**。

---

## 1. Spring AI 的定位

### 1.1 一句话定义

> Spring AI 是 Spring 官方推出的 AI 应用开发框架，把 LLM、Embedding、向量库等抽象成 Spring 风格的 Bean。

### 1.2 与 LangChain4j 的本质差异

| 维度 | LangChain4j | Spring AI |
|------|-------------|-----------|
| **设计哲学** | "用 Java 写 LangChain" | "把 AI 装进 Spring 生态" |
| **核心抽象** | `ChatModel`（接口） | `ChatClient`（Builder 风格 + Bean 注入） |
| **依赖注入** | 手动 `AiServices.builder()` | Spring `@Autowired` |
| **配置方式** | 代码 | `application.yml` + AutoConfig |
| **生态融合** | 独立运行 | 深度集成 Spring Boot |
| **官方背景** | 社区 | Spring 官方（VMware） |
| **AOP 机制** | 无（要自己装饰） | **Advisor 链**（Spring AOP 风格） |

### 1.3 什么时候选 Spring AI

✅ **选 Spring AI**：
- 项目本来就是 Spring Boot
- 团队熟悉 Spring 全家桶
- 需要 Spring Security / Cloud / Batch 等生态
- 想用配置文件管理多环境

✅ **选 LangChain4j**：
- 非 Spring 项目（CLI、库、Quarkus）
- 想要更细粒度的代码控制
- 学习 AI 概念（API 设计更"教科书"）
- 需要 LangChain Python 生态的概念映射

> **结论**：你是 Spring Boot 工程师，**生产环境大概率用 Spring AI**。

- **Spring AI 1.0 GA 的 starter 命名**：`spring-ai-starter-model-openai`（旧名 `spring-ai-openai-spring-boot-starter` 在 1.0 已重命名）
- **不需要 Ollama starter**：DeepSeek 和 LM Studio 都走 OpenAI 兼容协议，一个 starter 搞定 Chat + Embedding
- **Spring Boot 3.5.x** 是当前稳定版，Spring AI 1.0 GA 兼容良好

## 3. 配置 application.yml

本教程默认组合：**DeepSeek 远端 Chat + LM Studio 本地 Embedding**。两者都走 OpenAI 兼容协议，但 `base-url` 和 `api-key` 不同，Spring AI 支持通过 `chat.*` 和 `embedding.*` 子节点分别配置。

```yaml
spring:
  ai:
    openai:
      # Chat 走 DeepSeek
      base-url: https://api.deepseek.com
      api-key: ${DEEPSEEK_API_KEY}
      chat:
        options:
          model: deepseek-chat
          temperature: 0.7
      # Embedding 走本地 LM Studio（不同 base-url 和 api-key）
      embedding:
        base-url: http://127.0.0.1:1234
        api-key: lm-studio
        options:
          model: text-embedding-bge-large-zh-v1.5
```

**注意**：
- `base-url` 同样**不要带末尾斜杠**
- API Key 从环境变量读，不硬编码
- DeepSeek 不提供 Embedding 接口，所以 embedding 必须独立 base-url 指向 LM Studio
- 如果只用 Chat（不学 RAG），可以删掉 `embedding:` 那段

## 4. 第一次调用

### 4.1 用 ChatClient（推荐）

```java
package org.example;

import org.springframework.ai.chat.client.ChatClient;
import org.springframework.boot.CommandLineRunner;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.annotation.Bean;

@SpringBootApplication
public class Application {

    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }

    @Bean
    ChatClient chatClient(ChatClient.Builder builder) {
        return builder.build();
    }

    @Bean
    CommandLineRunner demo(ChatClient chatClient) {
        return args -> {
            String answer = chatClient.prompt()
                    .user("用一句话解释什么是 Agent")
                    .call()
                    .content();
            System.out.println(answer);
        };
    }
}
```

启动应用：
```bash
mvn spring-boot:run
```

### 4.2 关键认知

#### ChatClient.Builder 从哪来？

Spring AI 的 starter 自动装配了一个 `ChatClient.Builder` Bean，你直接注入即可。

**对比 LangChain4j**：
```java
// LangChain4j：手动 builder
ChatModel model = OpenAiChatModel.builder()
        .baseUrl("https://api.deepseek.com")
        .apiKey(System.getenv("DEEPSEEK_API_KEY"))
        .modelName("deepseek-chat")
        .build();

// Spring AI：Bean 注入
ChatClient client = chatClientBuilder.build();
```

#### `.prompt().user(...).call().content()` 链式调用

- `prompt()` —— 创建一个 prompt builder
- `user(...)` —— 加 user message
- `call()` —— 同步调用
- `content()` —— 提取返回的文本

类比 LangChain4j：
```java
// LangChain4j
String r = model.chat("hi");

// Spring AI
String r = chatClient.prompt().user("hi").call().content();
```

Spring AI 的链式更冗长，但**更灵活**（每一步都可以配置）。

## 5. 第一个 REST 接口

```java
@RestController
public class ChatController {

    private final ChatClient chatClient;

    public ChatController(ChatClient chatClient) {
        this.chatClient = chatClient;
    }

    @GetMapping("/hello")
    public String hello(@RequestParam(defaultValue = "你好") String q) {
        return chatClient.prompt()
                .user(q)
                .call()
                .content();
    }
}
```

启动后访问：
```
http://localhost:8080/hello?q=介绍一下Spring%20AI
```

## 6. 用原生 ChatModel（不用 ChatClient）

Spring AI 还有更底层的 `ChatModel` 接口：

```java
@RestController
public class RawController {

    private final ChatModel chatModel;

    public RawController(ChatModel chatModel) {
        this.chatModel = chatModel;
    }

    @GetMapping("/raw")
    public String raw(@RequestParam String q) {
        Prompt prompt = new Prompt(q);
        ChatResponse response = chatModel.call(prompt);
        return response.getResult().getOutput().getText();
    }
}
```

### 6.1 ChatModel vs ChatClient

| 维度 | ChatModel | ChatClient |
|------|-----------|-----------|
| 抽象层级 | 底层 | 高层 |
| 用法 | `call(Prompt)` | 链式 builder |
| Advisor 支持 | ❌ | ✅ |
| 默认 SystemMessage | ❌ | ✅ |
| 推荐度 | 仅特殊场景 | **默认用** |

**结论**：99% 场景用 `ChatClient`，只在需要极致控制时用 `ChatModel`。

类比 LangChain4j：
- Spring AI 的 `ChatModel` ≈ LangChain4j 的 `ChatModel`
- Spring AI 的 `ChatClient` ≈ LangChain4j 的 `AiServices` + `ConversationalChain` 的合体

## 7. 默认 SystemMessage 配置

```java
@Bean
ChatClient chatClient(ChatClient.Builder builder) {
    return builder
            .defaultSystem("你是公司客服助手，回答简洁专业。")
            .defaultTemperature(0.5)
            .build();
}
```

之后所有 `chatClient.prompt()` 都会自动带上这个 system message。

**对比 LangChain4j**：
```java
// LangChain4j：每个接口都要在 @SystemMessage 注解里写
@SystemMessage("你是客服")
String chat(String msg);
```

Spring AI 的"全局 default"更适合 Spring Boot 风格。

## 8. 常见错误

### 8.1 启动报错：找不到 ChatClient.Builder

**症状**：`No qualifying bean of type ChatClient$Builder`

**原因**：starter 没引对。
**解决**：确认 `spring-ai-starter-model-openai` 已在 pom，且 dependencyManagement 引了 BOM。

### 8.2 配置不生效

**症状**：用了 `application.yml` 的配置但没起效。

**排查**：
```bash
# 启动时加 --debug 看自动装配报告
mvn spring-boot:run -Dspring-boot.run.arguments="--debug"
```

搜 `SpringAI` 相关的 AutoConfiguration 是否生效。

### 8.3 LM Studio 连不上

排查同 LangChain4j 01 节。`lsof -i :1234` 看服务，浏览器访问 `http://127.0.0.1:1234/v1/models` 验证。

### 8.4 DeepSeek 404

`base-url` 不要带末尾 `/`。和 LangChain4j 同样坑。

### 8.5 找不到 milestone 仓库

```
Could not resolve org.springframework.ai:spring-ai-bom:1.0.0
```

确认 pom.xml 加了 `<repositories>` 里的 `spring-milestones`。

## 9. Spring AI vs LangChain4j 的设计哲学对比

### 9.1 装配方式

```
LangChain4j：                          Spring AI：
                                       
代码里写 Builder                       Spring 容器自动注入
ChatModel m = builder()...             @Bean ChatClient client(builder) {...}
AiServices.builder(Iface.class)        @Autowired ChatClient
        .chatModel(m)                  
        .tools(...)                    
        .build();                       
```

### 9.2 配置位置

```
LangChain4j：配置在代码里
Spring AI：配置在 application.yml
```

### 9.3 适合心智

| 你已有的心智 | Spring AI 对应 |
|------------|---------------|
| Spring Data JPA 的 Repository | ChatClient |
| Spring Security 的 Filter Chain | Advisor Chain |
| Spring Cloud 的 AutoConfiguration | spring-ai-xxx-starter |
| `@Bean` 注入 | ChatClient.Builder |
| `application.yml` | `spring.ai.*` 配置 |
