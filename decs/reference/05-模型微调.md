# 第四阶段 - 模型微调（Fine-tuning）

> 目标：让模型精通你的行话，解决提示工程和 RAG 无法实现的高难度场景。

---

## 1. 何时该微调（关键决策）

### 1.1 优先穷尽以下手段，再考虑微调

| 手段 | 覆盖需求比例 |
|------|------------|
| Prompt Engineering | 80% |
| Few-shot | +15% |
| RAG | 解决需要外部知识的问题 |
| **微调** | **最后 5%** |

### 1.2 该微调的场景
- 风格固化
- 特定格式遵循
- 领域术语
- 压缩 prompt 成本

### 1.3 不该微调的场景
- 知识更新频繁
- 数据量 <500 条
- 预算不支持反复迭代

---

## 2. 技术全景

| 技术 | 参数量 | 显存（7B） | 效果 | 何时选 |
|------|--------|----------|------|--------|
| 全参微调 (Full FT) | 100% | 100GB+ | 最好 | 极少用 |
| **LoRA** | 0.1~1% | 16GB | 接近全参 | **默认首选** |
| **QLoRA** | 0.1~1% | 6GB | 略逊 LoRA | **消费级显卡** |
| Prefix Tuning | 极小 | 8GB | 一般 | 已过时 |
| DPO / ORPO | 0.1~1% | 16GB | 对齐人类偏好 | 强化对齐阶段 |

---

## 3. LoRA 原理（必懂一句话）

> 冻结原模型权重，旁挂两个低秩矩阵 A(d×r) 和 B(r×d)，r 远小于 d。
> 训练时只更新 A、B，推理时合并回原权重。
> 数学基础是"低秩近似" —— 用低秩矩阵近似 ΔW。

## 4. 完整流程（按周）

### Week 1：数据准备（占工作量 60%）

#### 数据格式
```json
{
  "instruction": "评审以下 Java 代码",
  "input": "public void foo(){...}",
  "output": "未遵循规范 X..."
}
```

#### 数据来源
- 现有代码评审记录
- 人工标注
- 用 GPT-4 蒸馏（Cost 不可控，慎用）

#### 数量与质量
- 数量：500~10000 条起步
- **质量 >> 数量**

### Week 2：训练

#### 工具二选一

| 工具 | 特点 |
|------|------|
| **Llama-Factory**（`github.com/hiyouga/LLaMA-Factory`） | WebUI，零代码 |
| **PEFT 库 + Transformers**（`huggingface.co/docs/peft`） | 代码级控制 |

#### 关键超参

| 超参 | 推荐值 |
|------|--------|
| `lora_rank` | 8~64（小数据用 8，大数据用 32+） |
| `lora_alpha` | rank × 2 |
| `learning_rate` | 1e-4 ~ 3e-4 |
| `epochs` | 2~5（多了过拟合） |
| `target_modules` | `["q_proj","v_proj","k_proj","o_proj","gate_proj","up_proj","down_proj"]`（全量） |

### Week 3：合并 + 评估 + 部署

1. **合并 LoRA 权重到 base**：`peft.merge_and_unload()`
2. **用 vLLM 部署**合并后的模型
3. **评估**：
   - BLEU / ROUGE（弱）
   - 人工标注对比
   - **LLM-as-Judge**（用 GPT-4 打分）

1. **数据准备**：从公司内部 GitLab / SonarQue 抓取历史代码评审记录
2. **格式化**：`{"instruction":"评审代码","input":code,"output":review_comment}`
3. **微调**：Qwen2.5-Coder-7B（INT4，单卡 16GB 可跑）
4. **部署**：vLLM，集成进 GitLab CI（PR 自动评审）

## 7. 避坑点

- **灾难性遗忘**：微调后通用能力下降。**解决**：通用数据 + 领域数据混合训练。
- **过拟合**：epochs 别超 5。
- **数据泄漏**：评估集不能混入训练集。
- **学习率太大**：模型直接崩。LoRA 起步 lr 1e-4。
- **不要微调大模型做简单任务**：往往 prompt 就能解决。
