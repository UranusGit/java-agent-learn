# 附录 - 心智模型与决策树

> 给 Java 工程师的 30 分钟心智模型总结。

---

## A. 概念映射表

| Python/AI 概念 | Java 类比 |
|----------------|----------|
| LLM API | 一个有概率出错的远程 RPC |
| Prompt | RPC 的请求参数模板 |
| Function Calling / Tool | 注解声明的 RPC，LLM 帮你"决定调用哪个" |
| Tokenizer | 编码器（UTF-8 类比） |
| Embedding | 文本的哈希签名（保留语义） |
| 向量库 | 一个特殊的 B+ 树索引（按相似度查而不是精确匹配） |
| RAG | SQL JOIN 但条件是"语义匹配" |
| Agent | `while(true) { decide(); act(); observe(); }` 循环 |
| LangChain | AI 版的 Spring Framework |
| Fine-tuning | 给一个有经验的员工做岗位培训 |

---

## B. 决策树（何时用啥）

```
需求来了
  │
  ├── 简单问答 → Prompt Engineering (直接调 API)
  │
  ├── 需要外部知识 → RAG
  │     └── 精度要求高 → 混合检索 + 重排
  │
  ├── 需要调用系统 → Agent + Tool
  │     └── 多步推理 → LangGraph / Plan-Execute
  │
  ├── 风格/格式特殊 → Fine-tuning
  │
  ├── 多角色协作 → Multi-Agent
  │
  └── 海量领域数据 → Domain Pretraining
```

## C. 必读论文清单（按优先级）

1. **Attention Is All You Need** (2017) — Transformer 本体
2. **GPT-3 / InstructGPT** — LLM 范式
3. **ReAct** (2022) — Agent 思想
4. **Retrieval-Augmented Generation** (Lewis et al., 2020) — RAG 起源
5. **LoRA** (2021) — 微调
6. **Toolformer** (2023) — 工具学习
7. **PagedAttention** (vLLM, 2023) — 推理优化
8. **Constitutional AI** (Anthropic, 2022) — 对齐

---

## D. 防止迷失的"红线"

- **不要试图学会所有模型架构**：会用比会改重要 10 倍
- **不要陷在 Python 教程里**：你是 Java 工程师，**用 Spring AI / LangChain4j 实现一遍**才算掌握
- **不要追新框架**：每周都有新框架，盯死 LangChain4j + Spring AI 就够
- **每个项目都要有评估集**：没数字，就不叫工程，叫玄学
- **写文章输出**：博客/笔记是最佳学习加速器，比看 100 个视频有效

---

## E. 给 Java 工程师的特别建议

### E.1 心态是最大的优势
你把"服务挂掉"看成天大的事，这让你具备了让 AI Demo 走向生产的天然责任感。
别被一堆 PyTorch 代码吓住，**那只是另一个需要理解的库**。

### E.2 学习策略
**以项目为驱动，而不是以课程为驱动**。
每个阶段的学习，都对应一个能写进简历的 GitHub 项目。
你的 Java 经历也是宝藏 —— 把重构一个老旧 Spring Boot 服务加入 AI 能力的过程记录下来，**这就是独一无二的亮点**。
